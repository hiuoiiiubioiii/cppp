Deletion in an AVL Tree

Instructor: Jenny | Format: Continuous Study Feed

Sequence of nodes to delete: 8, 7, 11, 14, 17

The Two Golden Rules

Before touching the tree, the instructor establishes two fundamental rules for AVL Deletion. Keep these in mind at every step:

Rule 1: Delete like a Standard BST

Find the node and remove it using standard Binary Search Tree rules:

  • Leaf Node: Simply delete it.
  • One Child: Replace the node with its child.
  • Two Children: Replace the node with its In-order Predecessor (largest element in the left subtree) OR In-order Successor (smallest element in the right subtree).

Rule 2: Check Balance Factors (BF)

After deletion, recalculate the BF for every node, working from the deleted node's former position back up to the root.

  • Formula: BF = Height(Left) - Height(Right)
  • If BFs are -1, 0, or 1: The tree is balanced. Done.
  • If a BF is -2 or 2: You have a critical node! Apply AVL Rotations (LL, RR, LR, RL) to fix it.

Deleting Node 8

Locating: Start at root 14. 8 < 14 (Left to 11). 8 < 11 (Left to 7). 8 > 7 (Right to 8).

  • Visual Action: Node 8 is a leaf node (no children). It is simply deleted.
  • Balance Factor Check: We recalculate moving upwards.

    • Node 7: Left height is 1, Right is 0 ➔ 1 - 0 = 1
    • Node 12: Left is 0, Right is 1 ➔ 0 - 1 = -1
    • Node 11: Left height is 2, Right height is 2 ➔ 2 - 2 = 0

Status: Balanced

All Balance Factors are within [-1, 0, 1]. No rotation needed.

Deleting Node 7

Locating: Node 7 is the left child of 11.

  • Visual Action: Node 7 has one child (Node 4). Per BST rules, we erase 7 and connect Node 4 directly to Node 11.
  • Balance Factor Check: Recalculate upwards.
    • Node 4: 0
    • Node 11: Left height is 1 (Node 4), Right is 2 (path 12➔13) ➔ 1 - 2 = -1

Status: Balanced

All Balance Factors are within limits. No rotation needed.

Deleting Node 11 (Imbalance & Rotation)

Locating: Node 11 is the left child of the root 14.

  • Visual Action: Node 11 has two children (Left: 4, Right: 12). The instructor replaces it with the In-order Predecessor (largest value in the left subtree). Here, the left subtree only has Node 4. So, 4 replaces 11.

Crisis: Balance Factor Check

Recalculating BFs reveals an issue at the new Node 4:

  • Node 13: 0
  • Node 12: 0 - 1 = -1
  • Node 4: Left height is 0. Right height is 2 (path 12➔13) ➔ 0 - 2 = -2.

Because Node 4 has a BF of -2, it is a critical node. An imbalance has occurred!

The Rotation (RR Case): From critical node 4, the heavy path goes Right to 12, then Right to 13. This is an RR Imbalance, requiring One Left Rotation.

  • Node 12 gets pulled up to become the root of this sub-tree.
  • Node 4 gets pushed down to become the left child of 12.
  • Node 13 remains the right child of 12.

Deleting Node 14

Locating: Node 14 is the Root Node.

  • Visual Action: 14 has two children. We need the In-order Predecessor (largest value in the left subtree).
  • The left subtree is now headed by Node 12. The largest value in this left subtree is 13.
  • Erase root 14 and write 13 in its place. The old 13 is removed from the bottom.
  • Balance Factor Check:
    • Node 4: 0
    • Node 12: Left is 1, Right is 0 ➔ 1 - 0 = 1
    • Root Node 13: Left height is 2, Right is 3 ➔ 2 - 3 = -1

Status: Balanced

The new root stabilizes the tree. No rotation needed.
