Bubble Sort Algorithm

Complete Study Notes & Interactive Breakdown

1. The Basic Principle

Sorting means arranging data in a specific order (in this lesson: ascending order, from smallest to largest).

The Golden Rule of Bubble Sort:

Start from the beginning of the array. Compare two adjacent (side-by-side) elements. If the left element is greater than the right element, swap them. Otherwise, move to the next pair.

Why "Bubble"? Because after every complete pass through the array, the largest unsorted element "bubbles up" to its correct position at the far right end of the array.

3. Translating Logic into Code

From our trace, we noticed that for n = 5 elements, we needed exactly 4 passes to guarantee the array is sorted. Therefore, max passes = n - 1.

We need two loops: an outer loop for the passes, and an inner loop for the adjacent comparisons.

Attempt 1: The Unoptimized Code

// Outer loop runs n-1 passes
for (i = 0; i < n - 1; i++) {           
    
    // Inner loop checks every adjacent pair
    for (j = 0; j < n - 1; j++) {       
        if (A[j] > A[j+1]) {
            // Swap logic
            temp = A[j];
            A[j] = A[j+1];
            A[j+1] = temp;
        }
    }
}

The Flaw:

This code stupidly checks every element in the inner loop during every pass. But remember our visualizer? After Pass 1, the last element is already sorted. After Pass 2, the last two are sorted. We shouldn't compare them again!

4. Optimization #1: Fixing the Inner Loop

To stop comparing elements that are already "bubbled up" to their correct positions at the end of the array, we reduce the number of inner loop iterations as the outer loop (i) increases.

for (i = 0; i < n - 1; i++) {           
    // We subtract 'i' so we don't compare the elements already sorted at the end!
    for (j = 0; j < n - 1 - i; j++) {       
        if (A[j] > A[j+1]) {
            temp = A[j];
            A[j] = A[j+1];
            A[j+1] = temp;
        }
    }
}
  • When i = 0 (Pass 1): Loops up to n-1 (4 comparisons).
  • When i = 1 (Pass 2): Loops up to n-2 (3 comparisons).
  • When i = 2 (Pass 3): Loops up to n-3 (2 comparisons).

5. Optimization #2: The Flag Variable

What if the array is already sorted when we receive it? Or what if it gets fully sorted during Pass 2? The code above will blindly keep running passes 3 and 4 even though zero swaps happen.

The Fix: Introduce a boolean flag. If we go through an entire pass and make 0 swaps, the array is perfectly sorted, and we can break the outer loop early.

Final Optimized Bubble Sort Code:

for (i = 0; i < n - 1; i++) {
    
    flag = 0; // Reset flag at start of every pass
    
    for (j = 0; j < n - 1 - i; j++) {
        if (A[j] > A[j+1]) {
            temp = A[j];
            A[j] = A[j+1];
            A[j+1] = temp;
            
            flag = 1; // A swap happened! Set flag to 1
        }
    }
    
    // If no swaps happened in the inner loop, flag is still 0
    if (flag == 0) {
        break; // Array is sorted, exit early!
    }
}

6. Time Complexity Analysis

🟢 Best Case: O(n)

Scenario: Array is already sorted (e.g., [1, 2, 3, 4, 5]).

Why: The outer loop starts. The inner loop does one full pass of n-1 comparisons. Zero swaps occur. The flag remains 0, triggers the break, and the program ends immediately.

🔴 Worst Case: O(n²)

Scenario: Array is in strictly descending order (e.g., [5, 4, 3, 2, 1]).

Why: Every single adjacent pair requires a swap. Both the outer loop and inner loop must run to maximum capacity. Loop n times multiplied by loop n times = O(n²).
