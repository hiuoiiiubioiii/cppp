Topological Sort

Complete Video Masterclass Notes & Visualizations

📚 1. Definitions & Key Conditions

Topological sorting is an algorithm used heavily in scheduling tasks, resolving dependencies, and compiling code. Here are the whiteboard definitions you need to know:

The Definition

"It is a linear ordering of its vertices such that for every directed edge uv from vertex u to v, u comes before vertex v in the ordering."

Translation: If you have an arrow pointing from Node A to Node B, Node A must appear before Node B in your final sorted list.

The Must-Have Condition: DAG

The graph must be a DAG (Directed Acyclic Graph) for this to work.

  • Directed: Edges must have arrows showing a specific direction.
  • Acyclic: The graph must NOT contain any cycles (loops). If Node 1 points to 2, 2 points to 3, and 3 points back to 1, a topological sort is strictly impossible.

Multiple Solutions

Every DAG will have at least one topological ordering. However, a single graph can easily have multiple correct answers depending on which valid node you pick first.

⚙️ 2. The Core Algorithm (In-Degree Method)

Before jumping into examples, the instructor introduces the specific mathematical method used to solve these problems. It completely revolves around calculating degrees.

In-Degree The number of incoming arrows pointing at a specific node.
Out-Degree The number of outgoing arrows pointing away from a specific node.
The Golden Rule: To perform a topological sort, you must ALWAYS look for and select vertices that currently have an In-Degree of 0.

📝 3. Example 1: Full Step-by-Step Breakdown

Let's look at the first 5-node graph from the video. Here is the visual setup:

Step 1: Find Initial In-Degrees

We count how many arrows point into each node:

  • Node 1: 0 (Start here!)
  • Node 2: 1 (Arrow from 1)
  • Node 3: 2 (Arrows from 2, 4)
  • Node 4: 2 (Arrows from 1, 2)
  • Node 5: 1 (Arrow from 4)
Step 2: Begin the Sort

Select the node with an In-Degree of 0. That is Node 1.

Output: [1]

Update: Delete Node 1 and its outgoing edges (1→2, 1→4). Node 2 drops to In-Degree 0. Node 4 drops to In-Degree 1.

Step 3: Repeat

Node 2 now has an In-Degree of 0. Select Node 2.

Output: [1, 2]

Update: Delete Node 2 and its edges (2→3, 2→4). Node 4 drops to In-Degree 0. Node 3 drops to In-Degree 1.

Step 4: Repeat

Node 4 now has an In-Degree of 0. Select Node 4.

Output: [1, 2, 4]

Update: Delete Node 4 and its edges (4→3, 4→5). Node 3 drops to In-Degree 0. Node 5 drops to In-Degree 0.

Step 5: The Branching Point

Now, both Node 3 and Node 5 have an In-Degree of 0. Because it's a tie, you can pick either one. This creates multiple valid solutions.

Option A: Select 3, then 5.
[1, 2, 4, 3, 5]
Option B: Select 5, then 3.
[1, 2, 4, 5, 3]

⚠️ 4. Example 2: The Cyclic Graph Failure

The instructor draws this graph to prove exactly why a topological sort fails if the graph contains a cycle.

Notice the red curved line. There is a direct loop: 1 → 3 → 1.

Attempting the Sort

Initial In-Degrees: Node 4 0, Node 5 0. All others have > 0.

We safely select Node 4, then Node 5.

Output: [4, 5]
The Dead End

We delete 4 and 5. Look at the remaining nodes (1, 2, 3). Their updated In-Degrees are:

  • Node 1: 1
  • Node 2: 1
  • Node 3: 1

The algorithm breaks: We must select a node with an In-Degree of 0, but because of the cycle, none of the remaining nodes will ever drop to 0. You are permanently stuck.

💡 5. Example 3: Alphabetical Nodes

To solidify the concepts, the instructor does a final breakdown using letters.

Initial In-Degrees: A(0), C(0), D(2), B(3)

Because A and C both start at 0, we immediately branch into two valid cases.

Case 1: Start with A

  1. Select A. Delete edges.
  2. Update: C is 0, D drops to 1, B drops to 2.
  3. Select C. Delete edges.
  4. Update: D drops to 0, B drops to 1.
  5. Select D. Delete edge.
  6. Update: B drops to 0.
  7. Select B.
[A, C, D, B]

Case 2: Start with C

  1. Select C. Delete edges.
  2. Update: A is 0, D drops to 1, B drops to 2.
  3. Select A. Delete edges.
  4. Update: D drops to 0, B drops to 1.
  5. Select D. Delete edge.
  6. Update: B drops to 0.
  7. Select B.
[C, A, D, B]
