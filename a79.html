📚 B+ Tree Creation (Order = 5)

Instructor Notes & Visualizations

This feed contains a complete step-by-step breakdown of creating a B+ Tree, combining the instructor's verbal explanations with exact whiteboard replicas.


Data Array to Insert:

7, 10, 1, 23, 5, 15, 17, 9, 11, 39, 35, 8, 40, 25

⚙️ Step 1: Establish the Mathematical Rules

Before drawing anything, the instructor defines the constraints of the tree based on the Order (m) = 5.

  • Maximum Children: m = 5
  • Minimum Children: ⌈m/2⌉ = ⌈5/2⌉ = 3
  • Maximum Keys: m - 1 = 5 - 1 = 4 (This means a node splits when it gets a 5th key)
  • Minimum Keys: ⌈m/2⌉ - 1 = 3 - 1 = 2

🚨 The Root Node Exception

The instructor explicitly states that the Root Node is exempt from the minimum children and minimum key rules.

  • If it's the only node, it can have 0 children.
  • It is allowed to have only 2 children.
  • It is allowed to have only 1 key.

Note: The root MUST still obey the Maximum Key rule (max 4 keys).

🌳 Step 2: Initial Insertions & The First Split

Inserting: 7, 10, 1, 23

Because the maximum keys allowed is 4, we can fit all four of these numbers into a single sorted node.

171023

Inserting: 5 (Overflow!)

We want to insert 5, creating a temporary state of [1, 5, 7, 10, 23]. This exceeds the max keys (4). We must split from the median element.

  • The median of [1, 5, 7, 10, 23] is 7.
  • B+ Tree Leaf Rule: The median moves UP to become the parent, but a copy of it MUST REMAIN in the right-side leaf node.

🔥 Step 4: The "Double Split" (Crucial Concept)

After inserting 35, 8, and 40, our root node becomes completely full [7 | 9 | 15 | 23].

Now, we Insert 25. It goes to the rightmost leaf, creating an overflow: [23, 25, 35, 39, 40].

  1. First Split (Leaf Level): The median is 35. It pushes UP to the root. The leaf splits into [23 | 25] and [35 | 39 | 40].
  2. Second Split (Internal Node Level): The root receives 35 and overflows! Temporary state: [7, 9, 15, 23, 35]. We must split the root. The median is 15.

⚠️ Internal Node Split Rule

The instructor highlights a major difference here. When you split an internal node, the median moves UP to become the new parent, but it is NOT COPIED to the child node below it.

Internal nodes are just "indexes" or signposts. Actual data only lives in the leaves.

Median 15 moves up to form a brand new root node. Look at the resulting tree below:

📝 Exam Cheat Sheet: Key Takeaways

  • All Data in Leaves: In a B+ tree, every single inserted value must eventually appear in the leaf level.
  • Leaf Node Splits: The middle element moves UP to the parent, AND a copy stays in the right-side leaf node.
  • Internal Node Splits: The middle element moves UP to the parent, BUT DOES NOT stay in the child node. Internal nodes only hold indexes.
  • Root Exceptions: The root node is immune to minimum key and minimum child constraints. It can be as small as 1 key and 2 children.
  • Don't Forget the Arrows: Always draw horizontal arrows connecting your leaf nodes from left to right at the very end to prove it is a B+ tree, not a standard B-tree.
