1 Data Structures Overview

Whiteboard Notes:

Linear DS: Arrays, Linked List, Stack, Queue
Non-linear DS: Tree, Graph

🎙️ Instructor's Explanation

  • What is a Data Structure? It is a way to organize data in memory so it can be processed efficiently.
  • Linear Structures: Data is arranged sequentially (one after another). There is only one level. Elements are traversed in a single run (e.g., Stack follows LIFO, Queue follows FIFO).
  • Non-linear Structures: Data is arranged in a hierarchical manner with multiple levels. Data is not sequential. Trees and Graphs fall into this category.

  • Direction: Unlike a real tree that grows from the ground up, a Data Structure tree grows from the top to the bottom.

3 Formal Definition & Basic Terminology

  • A
    • B...
    • C
      • G
    • D...
Definition: "Tree can be defined as a collection of entities (nodes) linked together to simulate a hierarchy."

- Root = A
- Parent Node = C is parent of G
- Child Node = G is a child of C

🎙️ Instructor's Explanation

  • Node: Unlike arrays (which have "elements"), trees have "nodes." A node contains information and a link to the next node.
  • Root: The topmost node. It is the only node that does not have a parent.
  • Parent Node (C): The immediate predecessor (the node directly above) of any node.
  • Child Node (G): The immediate successor (the node directly below) of a node.

4 Advanced Terminology: Paths & Relationships

  • A
    • B
      • E
        • I
        • J
        • K

Leaf Node: I, J, K, F, L, M, H
Non-leaf Node: A, B, C, D, E, G
Path: sequence of consecutive edges from source to destination.
Ancestor: predecessor node on path to root.
Descendant: successor node on path to leaf.

🎙️ Instructor's Explanation

  • Leaf Node (I, K, F): A node that has zero children. It is at the very bottom of its path (also called an External Node).
  • Non-Leaf Node: A node that has at least one child (Internal Node).
  • Path (A→B→E→J): To get from A to J, you follow the connected edges. You count the edges (links) to determine path length.
  • Ancestor (Go Up): The ancestors of 'E' are B and A.
  • Descendant (Go Down): The descendants of 'B' are E, F, I, J, K.

5 Subtrees, Siblings, and Degree

🎙️ Instructor's Explanation

  • Subtree: Any node in the tree along with all of its descendants forms a completely valid smaller tree.
  • Sibling: Nodes that share the exact same parent. E and F are siblings. F and G are not siblings (they are cousins).
  • Degree of a Node: Just count its immediate children. Degree of A is 3. Degree of B is 2. Degree of any leaf node is always 0.
  • Degree of the Tree: Look at every node in the tree. Find the node with the highest degree. That number is the degree of the entire tree. (In this video's example, it is 3 because Node A has 3 children).

depth of node = length of path from root to that node
height of node = no. of edges in longest path from node to a leaf
Formula: A tree with N nodes has exactly (N - 1) edges.

🎙️ Instructor's Explanation

  • Depth (Looks UP): Count the edges from the ROOT down to your target node. Depth of Root(A) = 0. Depth of E = 2.
  • Height (Looks DOWN): Count the edges from your target node down to the deepest LEAF. Height of Leaf(J) = 0. Height of B = 2.
  • Height of the Tree: Equals the height of the Root node (which is 3).
  • Level: Represents horizontal layers. Rule of thumb: Level of a node = Depth of that node.
  • Edges Rule: A valid tree with N nodes will always have exactly N - 1 edges. If you add even one more edge, you create a cycle, and it is no longer a tree!

struct node {

    struct node *right;

🎙️ Instructor's Explanation

  • How do we actually code this? We use a Linked Representation (similar to doubly linked lists).
  • She introduces a Binary Tree (a tree where the maximum degree is 2).
  • Each node requires a struct with three components:
    1. Data: To store the actual value.
    2. Left Pointer: Memory address of the left child.
    3. Right Pointer: Memory address of the right child.
  • If a node is a leaf node, its left and right pointers will simply point to NULL.

8 Real-World Applications

  • 📁 File Systems: Your operating system (Windows, Linux) uses tree data structures to manage directories, sub-directories, and files efficiently.
  • 🌐 Networking: Routing protocols use tree data structures to find the shortest/best paths across networks.
  • 🔍 Searching & Sorting: Specifically formatted trees (like Binary Search Trees, AVL trees, Heap trees) are used when we need extremely fast, efficient data insertion, deletion, and searching operations.
