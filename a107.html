Radix Sort Algorithm

Instructor notes, whiteboard visualizations, and code breakdown

📌 1. Prerequisites & Core Concepts

Before diving into Radix Sort, the instructor heavily emphasizes that you must understand Counting Sort. Radix Sort does not sort elements on its own; it uses Counting Sort as a subroutine (a helper function) repeatedly.

How it sorts: Radix sort processes elements digit-by-digit.
  • For Integers: Sort from LSB to MSB (Least Significant Bit to Most Significant Bit). Meaning right-to-left (Ones, Tens, Hundreds).
  • For Strings/Names: Sort from MSB to LSB (Left-to-right).

Because Counting Sort is a stable sort, identical digits maintain their relative original order, which is what makes Radix Sort work correctly as we move to higher place values.

💻 6. Code Implementation

The instructor translates the whiteboard logic into C/C++ pseudo-code. Pay close attention to how we isolate the target digit using modulo arithmetic.

Key Math Trick: (A[i] / pos) % 10
If A[i] = 432 and we want the Tens place, pos is 10.
1. 432 / 10 = 43 (removes the ones place).
2. 43 % 10 = 3 (isolates the last digit, which is the tens place!) 
void RadixSort(int A[], int n) {
    // 1. Get the maximum element to find max digits
    int max = getMax(A, n); 

    // 2. Loop through each place value (pos = 1, 10, 100...)
    // As long as max/pos > 0, we still have digits to process.
    for (int pos = 1; max / pos > 0; pos = pos * 10) {
        // Call counting sort for the current digit place
        CountSort(A, n, pos);
    }
}

void CountSort(int A[], int n, int pos) {
    int count[10] = {0};  // Base 10 means indices 0-9
    int B[n];             // Output array

    // Step 1: Count frequency of specific digit
    for (int i = 0; i < n; i++) {
        ++count[(A[i] / pos) % 10];
    }

    // Step 2: Update count array to cumulative sum
    for (int i = 1; i <= 9; i++) {
        count[i] = count[i] + count[i - 1];
    }

    // Step 3: Build Output Array (Right to Left for stability)
    for (int i = n - 1; i >= 0; i--) {
        int digit = (A[i] / pos) % 10;
        B[--count[digit]] = A[i]; 
    }

    // Step 4: Copy sorted array B back into original array A
    for (int i = 0; i < n; i++) {
        A[i] = B[i];
    }
}

⏱️ 7. Complexity Analysis

At the end of the video, the instructor breaks down why Radix Sort is efficient but requires extra memory.

  • Time Complexity: O(d × (n + k))

    - n is the number of elements in the array.
    - k is the range of digits (for decimal, k = 10).
    - n + k is the time it takes for one pass of Counting Sort.
    - d is the number of digits in the maximum element (the number of passes).


  • Space Complexity: O(n + k)

    - We require an extra array `B` of size n to hold the sorted output during each pass.
    - We require a `count` array of size k (which is always 10).
