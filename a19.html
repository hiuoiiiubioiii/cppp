Reverse a Doubly Linked List

Complete Lecture Breakdown & Visualization Feed

1. The Core Concept

The goal of this algorithm is to reverse a doubly linked list (DLL). Before we write any code, we need to understand exactly what "reversing" means in this context.

  • What we are NOT doing: We are not physically moving the nodes in memory, nor are we swapping the data values inside them (e.g., swapping data 6 with 7).
  • What we ARE doing: We are simply reversing the links (pointers) between the nodes.

For every single node in the list, we need to swap the values of its prev and next pointers.

3. The Problem: Breaking the Chain

To reverse the links, we will take a pointer current to traverse the list starting at head.

Warning! If we immediately write current->next = current->prev, we run into a major issue!

Look at Node 1 (Address 200). Its next is currently 150. If we change its next to 0 (its `prev` value) right away, we lose the address 150 completely! The link is broken, and we can no longer move forward to traverse the rest of the list.

The Solution: We must use a second pointer, nextnode, to temporarily save the address of the next node before we overwrite the next pointer.

4. Writing the C Code

First, the instructor defines the structure and the global pointers:

struct node {
    int data;
    struct node *next;
    struct node *prev;
};

// Global pointers pointing to the start and end
struct node *head, *tail;

The Reversal Function

Here is the exact code block the instructor builds on the board step-by-step:

void reverseDLL() {
    // 1. Declare two extra pointers
    struct node *current, *nextnode;
    
    // 2. Start at the beginning
    current = head;
    
    // 3. Traverse the entire list
    while (current != 0) { // '0' represents NULL
        
        // Step 3a: Save the next node so we don't lose the list
        nextnode = current->next;
        
        // Step 3b: Swap the pointers in the current node
        current->next = current->prev;
        current->prev = nextnode;
        
        // Step 3c: Move the current pointer forward
        current = nextnode; 
    }
    
    // 4. Swap the global Head and Tail pointers
    current = head;
    head = tail;
    tail = current;
}

6. The Final Crucial Step: Swapping Head & Tail

If we stop after the loop, the individual links inside the boxes are perfectly reversed. However, head is still pointing to 200 and tail is pointing to 400!

If we try to print from head right now, the program goes to 200, sees that its next pointer is now 0, and stops immediately. It thinks the list is only one node long.

We must execute the final block of code to swap the global pointers:

current = head; // current = 200
head = tail;    // head = 400
tail = current; // tail = 200

Now, head correctly points to 400 (data 7), and tail correctly points to 200 (data 6).
