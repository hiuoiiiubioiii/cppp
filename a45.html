Circular Queue via Linked List

In-depth breakdown of instructor's whiteboard logic, code, and visualizations.

1. Why Do We Need a Circular Queue?

The Rule of Queues: They follow FIFO (First In, First Out). Insertions happen at the rear, and deletions happen at the front.

If we implement a standard Linear Queue using an array (e.g., size 5), a massive problem occurs. When the queue fills up, the rear pointer points to index 4. If we dequeue two items, indices 0 and 1 become empty. However, if we try to insert a new item, the queue says it's "Full" because the rear pointer is stuck at the end!

This results in memory wastage. The Circular Queue solves this by logically connecting the last index back to the first, creating a circle.

2. Transitioning to a Linked List

Using arrays limits our queue size. A Linked List allows our queue to grow dynamically. But how do we make it fast and circular?

  • The Problem: Standard linked lists only have a head. Deleting from the front is O(1), but traversing to the end to insert takes O(n) time.
  • The Solution: We use two pointers: front (for Dequeue) and rear (for Enqueue). This makes both operations O(1).
  • Making it Circular: Instead of the last node pointing to NULL, the rear node's next pointer connects back to the front node.

3. Global Setup & Node Structure

We start by defining the blueprint of our node and initializing the global pointers to `0` (NULL), indicating the queue is currently empty.

struct node {
    int data;
    struct node *next;
};

// Global pointers
struct node *front = 0;
struct node *rear = 0;

4. Enqueue Operation (Insertion)

To insert data, we first allocate memory. Then, we check if the queue is empty. If it is, the new node becomes both the front and rear, and points to itself. If it isn't, we add it after the current rear and update the circular link.

void enqueue(int x) {
    struct node *newnode;
    newnode = (struct node *)malloc(sizeof(struct node));
    newnode->data = x;
    newnode->next = 0;

    // Case 1: Empty Queue
    if(rear == 0) {
        front = rear = newnode;
        rear->next = front; // Circular link
    } 
    // Case 2: Multiple Nodes
    else {
        rear->next = newnode;
        rear = newnode;
        rear->next = front; // Maintain the circle
    }
}

5. Dequeue Operation (Deletion)

We always delete from the front. We must handle three scenarios: Underflow (empty), deleting the very last remaining node, and deleting from a list with multiple nodes.

void dequeue() {
    struct node *temp;
    temp = front; // Store to free later

    // Case 1: Queue is Empty
    if(front == 0 && rear == 0) {
        printf("Underflow!");
    }
    // Case 2: Only ONE node left
    else if(front == rear) {
        front = rear = 0;
        free(temp);
    }
    // Case 3: Multiple nodes exist
    else {
        front = front->next; // Move front forward
        rear->next = front;  // Rear points to new front
        free(temp);
    }
}

6. Peek & Display Operations

The Display Quirk: In a linear list, we loop until temp != NULL. Here, we loop until temp->next != front. Because the loop exits exactly at the last node, we must print the last node manually outside the loop!

void display() {
    struct node *temp = front;
    if(front == 0 && rear == 0) {
        printf("Empty");
    } else {
        while(temp->next != front) {
            printf("%d ", temp->data);
            temp = temp->next;
        }
        // Print the last node
        printf("%d ", temp->data);
    }
}

void peek() {
    if(front == 0) printf("Empty");
    else printf("%d", front->data);
}
