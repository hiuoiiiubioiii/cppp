1. The Basics: Standard Variables in Memory

Before understanding arrays, we must understand how a standard variable is stored in a computer's memory.

int a = 5;
  • This declares a single variable named a of integer data type.
  • In typical modern compilers, an int takes 4 bytes (32 bits) of memory.
  • The value `5` is converted to binary: 00000000 00000000 00000000 00000101 and stored across 4 consecutive bytes.

2. The Need for Arrays: The "Why"

Why do we need arrays when we have standard variables?

  • The Limitation: A standard variable can only hold one value at a time. If you write a = 6;, the 5 is overwritten and destroyed.
  • The Problem Scenario: Imagine you need to store the roll numbers of 60 students. Creating 60 separate variables (`int roll1, roll2...`) is highly inefficient and impossible to manage.
  • The Solution: An Array allows us to declare one variable name that can store multiple values.

3. Defining and Declaring Arrays

Definition: An array is a collection of more than one data item, but the data items must all be of the same data type.

Syntax in C/C++

DataType ArrayName [Size];

Example: int a[60]; (Stores 60 integers)

⚠️ Strict Compiler Rules for Size

  • Invalid: int a[]; (You cannot leave the size blank unless you initialize it immediately).
  • Invalid: int n; int a[n]; (The compiler must know the exact size at compile time. Variables whose values can change cannot be used here).

4. Array Initialization

There are two ways to put data into an array:

A. Compile-Time Initialization (Static)

You give the array its values directly in the code using curly braces.

int a[5] = {6, 2, 4, 3, 0};

Note: If you write int a[5] = {6, 2};, the remaining three spots will automatically be filled with 0s.

B. Run-Time Initialization (Dynamic)

You use loops to ask the user for input while the program is running.

int a[5];
printf("Enter the elements of array: \n");
for(int i = 0; i < 5; i++) {
    scanf("%d", &a[i]);
}

5. Under the Hood: Memory Representation

When you initialize int a[5] = {6, 2, 4, 3, 0};, how does it sit in RAM?

  • Memory Calculation: 5 integers × 4 bytes each = 20 total bytes needed.
  • Contiguous Allocation: The memory manager finds 20 consecutive empty bytes.

6. The Address Calculation Formula (Random Access)

Because arrays are stored contiguously, the computer does not have to search step-by-step for an element. It uses math to jump directly to it in constant time, known as O(1).

Formula: Address of a[i] = Base_Address + (i × Size_of_DataType)

Example Calculation

Let's find the memory address for a[2] from the visual above:

  • Base Address (address of a[0]) = 100
  • Target Index i = 2
  • Size of `int` = 4 bytes
Address = 100 + (2 × 4)
Address = 100 + 8
Address = 108

The computer jumps straight to address 108 to read the value '4'.

7. The Major Drawback: Fixed Size & Wasted Space

What happens if we allocate a large array but only put a few items in it?

char b[10] = {'j', 'e', 'n', 'n', 'y'};

  • Wasted Space: 10 bytes were requested, but only 5 are used. The rest are filled with null characters, wasting memory.
  • Lack of Flexibility: If you declare an array of size 100, but suddenly need to store 200 items at runtime, you cannot expand it. You must re-code and recompile.

8. Instructor's Final Board Summary

  • Definition: Array is a fixed-size sequenced collection of data items of the same data type.
  • Accessing: To access a particular element, use arrayname[index].
  • Storage: Data items are physically stored in continuous (contiguous) memory locations.
  • Speed: Allows Random Access, taking constant time O(1).
  • Types of Arrays: 1-D array (covered here), 2-D array (Matrices), Multi-dimensional array.
