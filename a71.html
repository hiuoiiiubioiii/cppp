🌳 1. What is a B-Tree?

Whiteboard Notes:
• balanced m-way tree (m = order)
• Generalization of BST
• node can have more than one key & more than 2 children
• maintains sorted data

The instructor starts by defining a B-Tree as a balanced m-way tree. The variable 'm' represents the order of the tree. If you are asked to build a B-Tree of order 5, then $m = 5$.

It is a generalization of a Binary Search Tree (BST). While a BST is restricted to 1 key per node and a maximum of 2 children, a B-Tree node can store multiple keys and branch out to multiple children.

📏 2. The Golden Level Rule

Whiteboard Note:
• all leaf node must be at same level

The instructor draws an unbalanced binary tree to illustrate a strict rule: In a standard BST, leaves can end at Level 2, Level 3, etc. In a B-Tree, this is strictly prohibited.

Every single leaf node (the nodes at the very bottom with no children) must terminate on the exact same horizontal level. This is what makes the B-Tree a perfectly "balanced" tree.

🧩 3. Anatomy of a B-Tree Node

Before doing the math, the instructor draws a large rectangular node to explain how data is stored.

  • Keys ($k_1, k_2, ...$): The actual data values stored inside the node.
  • Pointers/Links: Branches pointing downwards to child nodes. They sit in the spaces between and on the edges of the keys.

Crucial Concept: The number of child pointers coming out of a node is always exactly one greater than the number of keys inside that node. (e.g., 4 keys = 5 pointers).

👨‍👩‍👧‍👦 4. Mathematical Properties: Children

Whiteboard Notes:
B-tree of order m has following properties:
• Every node has max m children
• Min children:
    leaf → 0
    root → 2
    internal nodes → ⌈m/2⌉

The instructor breaks down the strict mathematical rules for how many children a node can have, based on order $m$. Let's use her example where $m = 5$.

  • Max Children: At most $m$. If $m=5$, maximum children = 5.
  • Min Children (Leaf): 0. (Leaves don't have children by definition).
  • Min Children (Root): 2. (Unless the tree is just one single node).
  • Min Children (Internal): ⌈$m/2$⌉. Internal nodes are nodes between the root and the leaves.
Instructor's Math Lesson: Ceiling Function ⌈ ⌉

m = 5
5 / 2 = 2.5
⌈2.5⌉ = 3 (Round UP to next integer)

Result: An internal node must have a minimum of 3 children.

🔑 5. Mathematical Properties: Keys

Whiteboard Notes:
• Every node has max (m-1) keys
• Min keys:
    root node → 1
    all other nodes → ⌈m/2⌉ - 1

Because the number of keys is always one less than the number of children, the key rules are directly derived from the children rules.

  • Max Keys: $m - 1$. If $m=5$, a node can hold a maximum of $5 - 1 = 4$ keys.
  • Min Keys (Root): 1.
  • Min Keys (All other nodes): Minimum children minus one. Since min children is ⌈$m/2$⌉, min keys is ⌈$m/2$⌉ - 1.
Math for m = 5

Min Children = ⌈5/2⌉ = 3
Min Keys = 3 - 1 = 2

Result: Every node (except root) must hold at least 2 keys.

⭐ 6. The Golden Rule of Insertion

At the very end of the video, the instructor states a vital rule for building B-trees that isn't written in the main bullet points, but is absolutely critical:

"The insertion would be always in the leaf node."

When you add a new value to a B-Tree, you never insert it directly into the root or an internal node. You traverse down the tree and place the new key into the appropriate leaf node. If that leaf node overflows (exceeds $m-1$ keys), it splits and pushes a value up. (She mentions this process will be covered in the next video).
