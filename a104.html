📚 Radix Sort

Complete In-Depth Study Notes & Visualizations

1. Introduction & Theoretical Concepts

Non-Comparison Sort: Unlike Bubble, Quick, Merge, Selection, or Insertion sorts (which rely on comparing elements directly using > or <), Radix Sort does not compare numbers against each other.

The Limit of Comparison Sorts: The best time complexity for comparison-based algorithms is O(n log n). They cannot mathematically perform better than this. Radix sort breaks this barrier by not using comparisons.

How Radix Sort Works: It sorts data digit by digit based on the place value (ones, tens, hundreds) or position of the digit.

Direction of Sorting:

Sorting happens from the LSD (Least Significant Digit) to the MSD (Most Significant Digit).

3. The Concept of "Buckets" (Base / Radix)

"Radix" means Base.

We are dealing with decimal numbers, so the base is 10. Therefore, we create 10 buckets numbered 0 through 9.

Side Note: If you were sorting alphabetical strings instead of numbers, the base is 26. You would use 26 buckets labeled from 0 to 25.

⏱️ Time Complexity Analysis

Radix Sort uses Count Sort as a subroutine (the act of putting numbers into buckets and pulling them out). The standard time complexity of Count Sort is O(n + b). Because Radix sort repeats this process d times, the final formula is:

Time Complexity = O(d * (n + b))

Variable Definitions:

  • d: The number of digits in the maximum number. This represents the number of times the outer loop runs (the number of passes). In our example, d = 3.
  • n: The total number of elements in the given array. In our example, n = 10.
  • b: The base (radix) of the number system. For decimal numbers, b = 10.
