Implementation of Circular Linked List in C

In-depth visual notes from the whiteboard lecture.

1. The Core Concept: Singly vs. Circular

The instructor begins by establishing what a standard Singly Linked List looks like. In a singly linked list, the final node always points to 0 (NULL), signifying the end of the list.

Singly Linked List (Before)

head
↓
200
200
6
150
➔
150
7
300
➔
300
1
0 (NULL)

How do we make it Circular?

To create a circle, we simply take the pointer of the very last node and replace the NULL value with the address of the first node (the head). In the whiteboard example, the head is at address 200.

Circular Linked List (After)

head
↓
200
200
6
150
➔
150
7
300
➔
300
1
200

Instructor Insight 💡

In a Circular Linked List, there is no "start" or "end" node just by looking at the circle. That is exactly why we must maintain a `head` pointer to keep track of our entry point into the circle.

2. Defining the Node Structure in C

Before writing the creation logic, we must define what a "node" looks like in code. Each node has two parts: a data part and a pointer part.

struct node {
    int data;
    struct node *next;
};

struct node *head = 0; // Global pointer initialized to NULL

Why struct node *next?

The instructor emphasizes this point heavily: If a pointer is going to store the address of an integer, it must be an int *. Because our next pointer is storing the address of another node, its data type must be a pointer to a struct node.

3. Creating the Circular Linked List (createCLL)

To dynamically create the list, we need to take input from the user in a loop, dynamically allocate memory using malloc, and correctly link the nodes.

The "Temp" Pointer Dilemma

When inserting nodes, you might be tempted to use the head pointer to link the new nodes together (e.g., head->next = newnode). Do not do this!

🚨 Common Student Mistake

If you move the head pointer to the newly created node, you lose the address of the very first node forever. You will never be able to access the start of your list. Always use a temp pointer to traverse and add nodes to the end!

The Full Creation Logic

Here is the exact code block written on the whiteboard, combining the memory allocation, the head check, the temp traversal, and closing the circular loop.

void createCLL() {
    struct node *newnode, *temp;
    int choice = 1;

    while (choice == 1) {
        // 1. Allocate dynamic memory for the new node
        newnode = (struct node*)malloc(sizeof(struct node));

        // 2. Populate the node with data
        printf("Enter data: ");
        scanf("%d", &newnode->data);
        newnode->next = 0; // Initialize next as 0 temporarily

        // 3. Link the node into the list
        if (head == 0) {
            // If list is empty, head and temp both point to the first node
            head = temp = newnode;
        } else {
            // Link the current last node (temp) to the new node
            temp->next = newnode;
            // Move temp forward to the new node
            temp = newnode;
        }

        // 4. MAKE IT CIRCULAR!
        // At the end of every insertion, the last node (temp) must point to head
        temp->next = head;

        // Ask user to continue
        printf("Press 1 to continue, 0 to exit: ");
        scanf("%d", &choice);
    }
}

Notice Step 4: temp->next = head;. Because temp is always updated to be the last node in the else block, we simply take temp->next and point it back to head to maintain the circle.

4. Displaying (Traversing) the List

Displaying a circular linked list requires a different while loop condition than a singly linked list.

The Infinite Loop Trap

In a singly linked list, we traverse using while (temp != 0). However, in a circular list, there is no 0 (NULL). If you use that logic, your code will print numbers in an infinite circle forever.

Instead, we must stop the loop when the next node is the head: while (temp->next != head).

void display() {
    struct node *temp;

    if (head == 0) {
        printf("List is empty");
    } else {
        temp = head; // Start at the beginning

        // Traverse the circle until the node points back to the head
        while (temp->next != head) {
            printf("%d ", temp->data);
            temp = temp->next;
        }

        // IMPORTANT: Print the very last node!
        printf("%d", temp->data);
    }
}

Why the extra printf at the end? 🤔

Look closely at the condition: while (temp->next != head). When temp reaches the last node, its next pointer does equal head. Therefore, the loop breaks before it has a chance to print the data inside the last node. We must manually write one final printf outside the loop to print that final value.

📝 Quick Summary Checklist

  • The Arrow Back: A Circular Linked List's final node points to the head, not NULL.
  • Never move Head: Use a temp pointer to add nodes to the end of the list. If you move head, the list is lost.
  • Closing the loop: After adding a node and moving temp, always write temp->next = head; to maintain the circle.
  • Traversal Condition: Use while (temp->next != head) to traverse, and remember to print the final node's data outside the loop.
