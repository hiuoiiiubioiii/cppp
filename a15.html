2. What is a Doubly Linked List (DLL)?

A Doubly Linked List is a linked list where each node contains two links instead of one. This splits the node into exactly three parts:

  • Prev Pointer: Stores the memory address of the preceding node.
  • Data Part: Stores the actual value (e.g., 7, 4, 9).
  • Next Pointer: Stores the memory address of the following node.

3. C Programming Representation

A node is a user-defined data type created using a struct. Because memory is dynamically allocated at runtime, we must declare a pointer (`head`) to access the first node.

Singly Linked List Code (For context)

struct node {
    int data;
    struct node *next;
};
struct node *head;

Doubly Linked List Code

To convert this into a DLL, we simply add one more pointer inside the struct to hold the previous node's address.

struct node {
    int data;
    struct node *next;
    struct node *prev; // New pointer for backward link
};
struct node *head;

4. Advantages of Doubly Linked Lists

Why do we use the extra memory for a DLL? It gives us massive advantages in traversal and modification.

  • Bidirectional Traversal: You can traverse the list forward AND backward.
  • Easy Reverse Printing: Traverse to the very end of the list, then simply use the `prev` pointers to print the data backwards.
  • Easier Deletion: You do not need to maintain a separate "trailing pointer" to delete a node, because the node itself knows the address of the node behind it.

The "Temp Pointer" Advantage

Imagine you have a pointer named temp currently pointing to the middle node (Address 200).

  • In an SLL, temp->next gets you to 400. But you are stuck. You cannot reach 100 without starting all over from the `head`.
  • In a DLL, temp->next gets you to 400, AND temp->prev gets you safely back to 100. Ultimate freedom!
