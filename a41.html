1. What is a Queue?

A Data Structure is a way of storing and organizing data in a computer. We have discussed Arrays, Linked Lists, and Stacks. Now, we introduce the Queue.

  • Linear Data Structure: Elements are arranged in a sequential order.
  • ADT (Abstract Data Type): In this lesson, we focus on what a queue is and what operations it can perform, rather than how it is coded under the hood (which comes later via Arrays/Linked Lists).

2. The Core Principle: FIFO

Real-Life Analogy: Imagine a movie ticket counter. The first person to stand in the line gets their ticket first. If you arrive after 5 people, you must stand at the 6th position.

F I F O

First In, First Out

Note: The instructor notes this can also be called LILO (Last In, Last Out). The last element added will be the last one to be removed. This is the exact opposite of a Stack (LIFO).

3. Rules & Terminology

Unlike a Stack (which has only one open end), a Queue follows strict rules based on two open ends.

  • Insertion (Adding data): Always happens at one specific end called the Rear (or tail). Operation: enqueue().
  • Deletion (Removing data): Always happens at the other specific end called the Front (or head). Operation: dequeue().

4. Fundamental Operations

To implement a Queue, we must maintain two pointers/variables: Front and Rear. Here are the 5 primary operations:

  • Enqueue(data): Requires an argument. Inserts data at the Rear.
  • Dequeue(): Requires no argument. Always removes the element currently at the Front.
  • front() or peek(): Returns the value of the element at the front without deleting it.
  • isFull(): Returns True if the queue has reached max capacity.
  • isEmpty(): Returns True if there are no elements in the queue.

6. Real-World Applications

Queues are used heavily in computer science and daily life, specifically when serving requests on a single shared resource.

🖨️
Shared Resources (Printers):

If 5 PCs send a print command simultaneously, the printer creates a queue. The first command received is processed first (FIFO), while the others wait in the queue.

🎧
Customer Care Phone Lines:

When you hear "Please hold," you are placed in a queue. The person who has been on hold the longest gets the next available representative.

💻
CPU Scheduling:

A CPU core executes one process at a time. The OS places waiting processes into a "Ready Queue." As soon as the CPU finishes a task, it fetches the next process from the front of the queue.
