The Whiteboard Rules

Before inserting data, you must memorize the three core properties of a Red-Black tree and the 4-step insertion algorithm written on the board.

3 Core Properties

  • 1. Root = Black: The top node must always be colored black.
  • 2. No two adjacent Red nodes: A Red parent cannot have a Red child. (Resolving this is the main goal of our algorithm).
  • 3. Black Depth: Every path from any node to a descendant `NULL` node must contain the exact same number of Black nodes.

The Insertion Algorithm

  • Step 1: If tree is empty, create newnode as root with color Black.
  • Step 2: If not empty, insert newnode (standard BST rule) as a leaf and color it Red.
  • Step 3: If parent of newnode is Black, exit. (No conflict).
  • Step 4: If parent is Red, check the Uncle (parent's sibling):
    • 4(a) If Uncle is Black or Null: Perform suitable AVL rotation (LL, RR, LR, RL) and recolor the new root and old root of that sub-tree.
    • 4(b) If Uncle is Red: Recolor Parent and Uncle to Black. Recolor Grandparent to Red. Recheck for conflicts starting from the Grandparent. (Note: Never recolor the absolute root to Red).
