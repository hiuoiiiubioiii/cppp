1. The Big Picture

Quick Sort is a sorting algorithm based on the Divide and Conquer technique. By default, our goal is to sort an array in ascending order.

The entire logic revolves around choosing a key element called the Pivot. While you can choose the last element, the median, or a random element, in this lecture, we always choose the very first element as the Pivot.

2. The Backbone: "Partitioning"

The instructor emphasizes: "If you know how to partition the array, you know how to write the Quick Sort code."

What is Partitioning? It is the process of taking the Pivot element and placing it into its exact, final sorted position inside the array.

⚠️ Crucial Concept: After partitioning, the elements in Partition 1 and Partition 2 are NOT sorted among themselves. The only guarantee is that all numbers to the left are smaller than the pivot, and all numbers to the right are larger. Quick sort will then recursively do this same process to the left and right partitions.

3. The Rules of the Pointer Game

To partition an array in code, we use two pointers: start and end.

  • The start pointer: Begins at the Lower Bound (index 0) and moves right (→). It looks for an element that is greater than the Pivot. (It stops when it finds one).
  • The end pointer: Begins at the Upper Bound (last index) and moves left (←). It looks for an element that is less than or equal to the Pivot. (It stops when it finds one).
  • The Action: When both pointers stop, you swap the elements they are pointing at.
  • The End Game: If the start and end pointers cross each other (start > end), the game stops! You swap the Pivot with the element at the end pointer.

5. The Code: Partition Function

Here is the code written on the whiteboard representing our pointer game.

int Partition(A, lb, ub) {
    pivot = A[lb];  // Pick first element
    start = lb;
    end = ub;
    
    while (start < end) { 
        
        // Move start to the right
        while (A[start] <= pivot) { 
            start++; 
        }
        
        // Move end to the left
        while (A[end] > pivot) { 
            end--; 
        }
        
        // If they haven't crossed, swap them
        if (start < end) { 
            swap(A[start], A[end]); 
        }
    }
    
    // Pointers crossed! Swap pivot with 'end'
    swap(A[lb], A[end]);
    
    // Return the sorted location of the pivot
    return end; 
}

6. The Code: Quick Sort Algorithm

This is the main recursive function that drives the entire algorithm.

QuickSort(A, lb, ub) {
    // Base Case: Proceed if there are >= 2 elements
    if (lb < ub) { 
        
        // 1. Partition array and get pivot's final index
        loc = Partition(A, lb, ub); 
        
        // 2. Sort the LEFT sub-array
        QuickSort(A, lb, loc - 1);
        
        // 3. Sort the RIGHT sub-array
        QuickSort(A, loc + 1, ub); 
    }
}

7. Time Complexity Summary

To conclude the lecture, the instructor outlines the time complexity of Quick Sort (to be detailed mathematically in a future video).

Worst Case

O(n²)

Occurs when the array is already completely sorted or completely reverse-sorted, and you always pick the first/last element as the pivot.

Best / Average Case

O(n log n)

Occurs when the pivot relatively evenly divides the array into halves. Quick Sort generally outperforms Bubble, Selection, and Insertion sorts.
