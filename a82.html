⏱️ The Time Variable Matrix

During a Depth First Search (DFS), we use a global time variable to track our journey. The instructor heavily relies on this to classify edges.

For every node, we record two specific timestamps:

  • Start/Discovery Time: The exact moment we arrive at a brand new node. The instructor refers to this as the moment a node "appears".
  • Finish Time: The exact moment we have fully explored all paths from a node and are forced to backtrack.

🗂️ The Four Types of Edges

When looking at an edge e:(x, y) pointing from node X to node Y, we can categorize it into one of four types based on its timestamps and path structure.

1. Tree Edge

An edge that is an actual member of the DFS traversal. You use this edge to discover a node for the very first time.

2. Forward Edge

Node Y appears after Node X.
Condition: StartTime(x) < StartTime(y)
Requirement: There must be a path from X down to Y through the DFS tree.

3. Back Edge

Node Y appears before Node X.
Condition: StartTime(y) < StartTime(x)
Requirement: There must be a path from Y down to X. (This creates a cycle!)

4. Cross Edge

Node Y appears before Node X, BUT they belong to different branches.
Requirement: There is NO path from Y to X.

🔍 Evaluating the Non-Tree Edges

Once all Tree Edges are marked, we evaluate the remaining edges using our table.

Example 1: Edge (d, b)

x = d (Start: 6), y = b (Start: 2)

y appeared before x. Is there a path from y to x? Yes (b → c → d).

Result: Back Edge

Example 2: Edge (g, m)

x = g (Start: 11), y = m (Start: 20)

y appeared AFTER x. Is there a path from x to y? Yes (g → h → k → m).

Result: Forward Edge

Example 3: Edge (d, e)

x = d (Start: 6), y = e (Start: 4)

y appeared before x. Is there a path from y to x (e → d)? NO.

Result: Cross Edge
