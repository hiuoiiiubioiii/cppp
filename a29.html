1. Core Concept & Logical Rotation

A stack can be implemented using Arrays (Static Memory) or Linked Lists (Dynamic Memory). Here, we focus entirely on the Array implementation.

Memory Math

When you declare an array like int a[5];, the computer allocates 5 contiguous blocks in memory.

  • If 1 integer takes 4 bytes...
  • 5 integers will take exactly 20 bytes of memory.
  • Indexes run from 0 to 4.

2. Global Declarations

Because we are building a menu-driven program where multiple functions (Push, Pop, Peek, Display) need access to the exact same stack, we declare our variables globally (outside of `main`).

// Global Declarations
#define N 5
int stack[N];
int top = -1;

Instructor's Breakdown:

  • #define N 5: This is a macro. It sets the max size to 5. Using a macro makes it incredibly easy to change the stack size later without hunting through your code.
  • int top = -1: Crucial variable. `top` tracks the index of the highest element. It starts at `-1` because the stack is currently empty (if it started at 0, that would mean there is an element at index 0!).

4. The push() Operation

Goal: Check for overflow, increment the top pointer, and insert data.

void push() {
    int x;
    printf("Enter data: ");
    scanf("%d", &x);

    if (top == N - 1) {
        printf("Overflow");
    } else {
        top++;
        stack[top] = x;
    }
}

Instructor's Breakdown:

  • Check Overflow: We must check if the stack is full. Since max size is N (5), the highest index is 4 (N - 1). If top == N - 1, we print "Overflow".
  • Increment FIRST: If there is space, we do top++. If `top` was -1, it becomes 0.
  • Insert LAST: Now that `top` points to a valid empty slot, we insert: stack[top] = x;.

5. The pop() Operation

Goal: Check for underflow, retrieve the top value, and decrement the top pointer.

void pop() {
    int item;
    if (top == -1) {
        printf("Underflow");
    } else {
        item = stack[top];
        top--;
        printf("The popped item is %d", item);
    }
}

Instructor's Breakdown:

  • Check Underflow: If top == -1, the stack is completely empty. Print "Underflow".
  • The Garbage Value Concept: Look closely at top--;. Notice we don't actually "delete" the memory block. We just lower our pointer. The old data is left behind as "garbage" and will simply be overwritten the next time we push data.

6. The peek() (or top) Operation

Goal: Look at the top element without modifying the stack.

void peek() {
    if (top == -1) {
        printf("Stack is empty");
    } else {
        printf("The topmost element is %d", stack[top]);
    }
}

This is almost identical to pop(), but we do not write top--;. The pointer stays exactly where it is.

7. The display() Operation

Goal: Traverse the stack and print elements from top to bottom (LIFO order).

void display() {
    int i;
    for (i = top; i >= 0; i--) {
        printf("%d, ", stack[i]);
    }
}

Because stacks follow LIFO (Last In, First Out), we must print the top item first. Therefore, the for loop initializes i = top and counts downwards (i--) until it hits the bottom index (i >= 0).

8. The main() Function Menu

Goal: Tie it all together using a Switch case inside a Do-While loop.

void main() {
    int ch;
    do {
        printf("Enter choice: 1:Push 2:Pop 3:Peek 4:Display 0:Exit\n");
        scanf("%d", &ch);

        switch (ch) {
            case 1: push(); break;
            case 2: pop(); break;
            case 3: peek(); break;
            case 4: display(); break;
            case 0: printf("Exiting..."); break;
            default: printf("Invalid choice");
        }
    } while (ch != 0);
}

Important Notes:

  • The do-while loop ensures the menu prints to the user at least once.
  • The instructor explicitly stresses the importance of break;. If you forget to write break; after calling push(), the code will "fall through" and automatically execute pop() without asking the user!

9. Time Complexity Summary

At the end of the lecture, the instructor summarizes the efficiency of an array-based stack:

  • Push: O(1) - Constant time. No loops, just basic math and assignment.
  • Pop: O(1) - Constant time. Just decreasing the top pointer.
  • Peek/Top: O(1) - Constant time. Directly accessing an array index.
  • Display/Search: O(n) - Linear time. Requires a for-loop to traverse through 'n' elements.
