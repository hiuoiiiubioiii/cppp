Topic Overview

The Bellman-Ford Algorithm

Welcome! This feed contains your complete, in-depth study notes for the Bellman-Ford Algorithm. Keep scrolling to learn how it works, step-by-step.

What is it?

It is a Single-Source Shortest Path algorithm. It finds the shortest distance from one starting vertex (the source) to all other vertices in a weighted, directed graph.

Bellman-Ford vs. Dijkstra's Algorithm

  • Dijkstra's Algorithm is generally faster, but it fails (or gives incorrect answers) if the graph contains negative edge weights.
  • Bellman-Ford guarantees the correct answer even if the graph has negative edge weights.
  • The Trade-off: Bellman-Ford is slower. You should use Dijkstra's for positive-only graphs, and Bellman-Ford when negatives are involved.
Core Concept

The Golden Rule

The entire algorithm is built on one simple, repetitive principle.

"Go on relaxing all the edges (n - 1) times."

Where n = the total number of vertices in the graph.

Why exactly (n - 1) times?

In the absolute worst-case scenario, a graph is a straight line. The shortest path between the first node and the last node can contain at most n - 1 edges. Repeating the process n - 1 times guarantees that the shortest path has enough "time" to propagate to the furthest node.

The Math

The Relaxation Formula

"Relaxing" an edge is just a fancy term for asking a simple question: "Have I found a shorter path to this node?"

  • Let u = Starting node of an edge
  • Let v = Destination node of an edge
  • Let d[] = Array storing the current shortest distance to a node
  • Let c(u, v) = The cost/weight of the edge between u and v
if (d[u] + c(u, v) < d[v]) { d[v] = d[u] + c(u, v); }

Translation: If the distance to node u plus the weight of the edge to v is strictly less than the currently known distance to v, update v's distance to the new, smaller number.

Analysis

Time Complexity Breakdown

How long does Bellman-Ford take to run?

O(E × V)

The Math: We check every Edge (E), and we repeat this process V - 1 times (where V is vertices). Thus, E × (V - 1) simplifies to O(E × V).

The Worst-Case Scenario

In a "complete graph," every vertex connects to every other vertex. The maximum number of edges E becomes roughly V².

Substitute V² for E in our formula:
O(V² × V) = O(V³)

The worst-case time complexity is O(V³) (or O(n³)).

The Major Drawback

Negative Weight Cycles

Bellman-Ford handles negative edges perfectly, BUT it completely breaks if the graph contains a Negative Weight Cycle.

A negative cycle is a closed loop where the sum of the edge weights equals a negative number. Let's look at the cycle drawn by the instructor:

Cycle Math: 7 + (-15) + 7 = -1

Why does this break the algorithm?

If you keep circling through this loop, the total distance drops by 1 every single time. It will keep getting smaller towards negative infinity. Therefore, an actual "shortest" path doesn't exist.

How Bellman-Ford Detects This:

To detect a negative cycle, the algorithm does a clever trick:

  1. Run the standard algorithm n - 1 times.
  2. Run it ONE extra time (the nth iteration).
  3. If any distance still gets smaller during this extra loop, it proves a negative cycle exists, and the algorithm throws an error.
