Study Feed: Minimum Spanning Trees (MST)

Topic: Modifying Edge Weights (Squaring weights) | Continuous Scroll

1. The Problem Statement

We are given a scenario with two Minimum Spanning Trees constructed from the exact same vertices, but with different edge weight rules.

Let x = cost of MST G(V, E)
Let y = cost of MST G(V, E²)

Where E² means:
If weight w(u,v) ∈ E, then w²(u,v) ∈ E²

Question: What is the relationship between x and y?

⚠ CRUCIAL INSTRUCTOR HINT WRITTEN ON BOARD:
"w ∈ Real number"

This box drawn on the board changes everything. Edge weights are NOT limited to positive whole integers. They can be decimals, fractions, etc.

  • (i) x ≥ y
  • (ii) x ≤ y
  • (iii) x = y
  • (iv) None of these

2. Setting up the Approach & Graph Properties

The instructor decides to test the problem using a counter-example approach. She starts by drawing a simple 4-node path graph.

Core MST Rule Mentioned: If an MST has N vertices, it will always have exactly N-1 edges.

Because the problem states both graphs are built on G(V...), both graphs have the exact same number of vertices (V). Therefore, both graphs will have exactly 3 edges. If graph 2 had V' (different vertices), the problem would be unsolvable.

3. Case Study 1: Standard Integers (> 1)

The instructor assigns random integer weights to Graph 1, and squares them to create Graph 2.

Graph 1 (Cost = x)

5
3
2
x = 5 + 3 + 2 = 10

Graph 2 (Cost = y)

25
9
4
y = 25 + 9 + 4 = 38

Result of Case 1: Here, x (10) < y (38). A student might rush and pick option (ii) x ≤ y. The instructor warns this is a trap! We must test the "Real Numbers" constraint.

4. Case Study 2: The Trap (Decimals between 0 and 1)

To test the "Real Number" condition, she erases the integers and replaces them with decimal values.

Graph 1 (Cost = x)

0.5
0.3
0.2
x = 0.5 + 0.3 + 0.2 = 1.0

Graph 2 (Cost = y)

0.25
0.09
0.04
y = 0.25 + 0.09 + 0.04 = 0.38

Result of Case 2: Squaring a number between 0 and 1 makes it smaller. Now, x (1.0) > y (0.38). The relationship has flipped completely.

5. Final Conclusion

Because the relationship changes depending on whether the original weights are greater than 1 or between 0 and 1, there is no absolute, constant mathematical relationship between x and y.

Case 1: x < y
Case 2: x > y
Therefore, no single rule applies.
  • (i) x ≥ y
  • (ii) x ≤ y
  • (iii) x = y
  • ✔ (iv) None of these.

💡 Key Takeaway: In competitive exams (GATE, NET, etc.), always look for constraints like "Real numbers." If weights aren't explicitly restricted to integers > 1, always test a counter-example with a decimal (0 to 1) or a negative number before locking in your answer.
