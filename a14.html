Reverse a Linked List

An In-Depth Iterative Approach Using C

1 Core Concept

There are two ways to reverse a linked list: Iteratively (using loops) and Recursively (using function calls). This feed focuses entirely on the Iterative approach.

Common Misconception!

Reversing a linked list does not mean swapping the data values inside the nodes (e.g., physically moving 5 to the end). Instead, it means reversing the links (pointers) between the nodes.

2 The "Broken Link" Dilemma

If we are standing at Node 1 (address 100) and we immediately change its next pointer from 200 to 0, we lose the rest of the list! We no longer have the address 200 to move forward.

The 3-Pointer Solution

To traverse and reverse without losing links, we need three separate pointers:

prevnode

Tracks the node behind us. We need this to point our current node backwards.

currentnode

The node we are actively modifying right now.

nextnode

Safely secures the rest of the list before we break the link.

3 The C Code Logic

Here is the prerequisite structure and the full logic written out.


// Prerequisites globally declared
struct node {
    int data;
    struct node *next;
};
struct node *head;

// The Reverse Function
void reverse() {
    struct node *prevnode, *currentnode, *nextnode;
    
    prevnode = 0;
    currentnode = nextnode = head;
    
    while (nextnode != 0) {
        nextnode = nextnode->next;       // Step A: Secure the next node
        currentnode->next = prevnode;    // Step B: Reverse the link!
        prevnode = currentnode;          // Step C: Move prev forward
        currentnode = nextnode;          // Step D: Move curr forward
    }
    
    head = prevnode;                     // Step E: Update global head
}

Conclusion & Edge Cases

Because of the way the while loop is constructed, this code handles edge cases flawlessly. If you pass in a linked list with only one node, nextnode immediately becomes 0 on Iteration 1, the link is updated to 0, and head updates successfully without breaking.
