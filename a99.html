Selection Sort Algorithm

1. Initial Setup & Concept

Selection Sort works by logically dividing the array into two parts: a Sorted Sub-array and an Unsorted Sub-array. Initially, the sorted part is empty, and the unsorted part contains all elements.

Target: Sort in Ascending Order.

The Core Logic

  • Look at the Unsorted Sub-array.
  • Find the minimum (smallest) element in this unsorted section.
  • Swap that minimum element with the element located at the starting position of the unsorted sub-array.
  • Move the boundary of the sorted array one step to the right. Repeat.

Crucial Logic Point: Notice that we only needed 5 passes for an array of 6 elements. Once 5 elements are sorted, the 6th element (10) is automatically left in its correct, sorted position.

Rule: Number of passes required = n - 1.

3. C/C++ Code Implementation

Pay close attention to the variable min. It stores the index of the minimum element, not the actual value.

// Outer loop controls the number of passes.
// It runs from 0 to n-2 (which is < n-1)
for (int i = 0; i < n - 1; i++) 
{
    // Assume the first element of the unsorted part is the minimum
    int min = i; 
    
    // Inner loop searches the unsorted part for a smaller element
    for (int j = i + 1; j < n; j++) 
    {
        // If we find an element smaller than our current minimum...
        if (a[j] < a[min]) 
        {
            // Update 'min' to hold the index of this new smallest element
            min = j; 
        }
    }
    
    // Optimization check: If the minimum element is already in the right place, 
    // don't bother swapping.
    if (min != i) 
    {
        swap(a[i], a[min]);
    }
}

4. Dry Run / Tracing the Code

Let's prove the code works by tracing variables on the board for Pass 1 (i = 0).

Initial State

  • i = 0 // First pass begins
  • min = 0 // Assume a[0] (7) is smallest
  • j = i + 1 = 1

Inner Loop Execution

  • j=1: a[1]<a[min] ? 4<7? YES → min = 1
  • j=2: a[2]<a[min] ? 10<4? NO
  • j=3: a[3]<a[min] ? 8<4? NO
  • j=4: a[4]<a[min] ? 3<4? YES → min = 4
  • j=5: a[5]<a[min] ? 1<3? YES → min = 5

After Inner Loop:

Check optimization: if (min != i) → if (5 != 0) → True.

Execute: swap(a[0], a[5]).
Array [7, 4, 10, 8, 3, 1] becomes [1, 4, 10, 8, 3, 7].
Pass 1 is complete!

5. Time Complexity Analysis

O(n²)

Why is it O(n²)?

  • The outer loop i runs approximately n times.
  • The inner loop j runs n-1, then n-2, then n-3 times, scaling geometrically.
  • Unlike Insertion Sort or Bubble Sort, Selection Sort does not care if the array is already sorted.
  • Even if you give it a perfectly sorted array like [1, 2, 3, 4, 5], the inner loop must still completely scan all remaining elements just to verify that no smaller element exists.

O(n²)
