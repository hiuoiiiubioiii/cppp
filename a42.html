Implementation of Queue Using Arrays

Complete Video Breakdown & Study Notes

1. Queue Fundamentals

A Queue is a linear data structure that follows a specific logical rule:

  • FIFO Rule: First In, First Out. The first element added is the first one to be removed.
  • Insertion (Enqueue): Happens exclusively at the Rear end.
  • Deletion (Dequeue): Happens exclusively at the Front end.
  • Time Complexity: Both Enqueue and Dequeue operations must execute in O(1) time.

2. Global Setup & Memory Allocation

We use Static Memory Allocation via an array. To ensure all functions can access the queue without constantly passing variables, we declare the array and our pointers globally.

#define N 5          // Macro: 'N' is replaced by 5 everywhere

int queue[N];        // Declare array of size 5
int front = -1;      // Front pointer initialized to -1 (empty)
int rear = -1;       // Rear pointer initialized to -1 (empty)

3. Enqueue Operation (Insertion)

Inserting an element x happens at the rear. We must check 3 exact conditions:

  • Condition 1 (Overflow): rear == N - 1. If rear has reached the end of the array, we cannot insert.
  • Condition 2 (First Element): front == -1 && rear == -1. If the queue is totally empty, we must move both pointers to index 0 before inserting.
  • Condition 3 (Normal): Simply increment rear++ and place the new value there.
void enqueue(int x) {
    if (rear == N - 1) {
        printf("Overflow");
    } 
    else if (front == -1 && rear == -1) {
        front = rear = 0;
        queue[rear] = x;
    } 
    else {
        rear++;
        queue[rear] = x;
    }
}

4. Dequeue Operation (Deletion)

Removing an element happens at the front. We must check 3 exact conditions:

  • Condition 1 (Underflow): front == -1 && rear == -1. The queue is empty, nothing to delete.
  • Condition 2 (Only 1 element left): front == rear. Deleting this empties the queue entirely. We must reset both pointers to -1.
  • Condition 3 (Normal): Print the value, then increment front++. (The old data remains in memory as "garbage" but the queue logically ignores it).
void dequeue() {
    if (front == -1 && rear == -1) {
        printf("Underflow");
    } 
    else if (front == rear) {
        printf("Dequeued: %d", queue[front]);
        front = rear = -1;
    } 
    else {
        printf("Dequeued: %d", queue[front]);
        front++;
    }
}

5. Auxiliary Functions (Display & Peek)

Display()

We traverse the array starting from the front index up to the rear index.

void display() {
    if (front == -1 && rear == -1) {
        printf("Queue is empty");
    } else {
        for(int i = front; i < rear + 1; i++) {
            printf("%d ", queue[i]);
        }
    }
}

Peek()

Returns the value at the front of the queue without removing it.

void peek() {
    if (front == -1 && rear == -1) {
        printf("Queue is empty");
    } else {
        printf("%d", queue[front]);
    }
}

6. The Major Drawback of Linear Arrays

At the end of the lecture, a massive flaw is revealed. Imagine a scenario where you enqueue elements until the array is full, and then dequeue several elements.

If we try to enqueue now, the code checks: if (rear == N - 1).
Since rear is at 4, it prints "Overflow"!

Even though indices 0, 1, and 2 are perfectly empty and available, we cannot insert data because the rear pointer is mathematically stuck at the end of the array. This is a severe waste of memory.

The Solution Introduced: Circular Queue

To fix this, we stop using rear++. Instead, we use the modulo operator formula:

rear = (rear + 1) % N

This formula forces the pointer to "wrap around" back to index 0 if it hits the end of the array, successfully utilizing the wasted spaces. This leads into the topic of Circular Queues!
