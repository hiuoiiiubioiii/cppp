Double Hashing Technique

Complete video breakdown & visual study notes.

1. Initial Setup & Given Data

Before we begin inserting, let's map out the problem constraints written on the whiteboard:

Array of Keys (A): [3, 2, 9, 6, 11, 13, 7, 12]
Hash Table Size (m): 10 (Indices 0 to 9)
Primary Function (h₁): h₁(k) = 2k + 3
Secondary Function (h₂): h₂(k) = 3k + 1

2. The Core Formula

In Double Hashing, we insert the key k at the first free place using this specific formula when a collision occurs:

Location = (u + v * i) % m
  • u (Initial Location): Calculated using the first hash function. u = h₁(k) % m
  • v (Step Size): Calculated using the second hash function. Only calculate this if a collision happens! v = h₂(k) % m
  • i (Probe Number): The attempt number. Starts at 0, then 1, 2, etc.

4. Final Exam Question & Output

At the end of the video, the instructor explains how to answer a standard exam question: "What is the final order of elements in the hash table?"

You read the hash table top to bottom (index 0 to 9), using a dash (-) for empty slots.

Final Output Array:
-, 9, -, 11, 12, 6, -, 2, -, 3

Instructor's Final Insight 💡

Notice that keys 13 and 7 failed to insert, even though there were empty spaces remaining. This highlights that the provided hash functions were flawed for this dataset. A "good" double hash function must ensure that v ≠ 0 and that v is relatively prime to m so it checks every single slot before looping.
