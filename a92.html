Dijkstra's Algorithm

Complete Video Lecture Breakdown & Visual Study Guide

🎯 Part 1: The Core Concept

Dijkstra’s Algorithm is used to solve the Single Source Shortest Path problem. This means you are given one starting point (the source node), and your goal is to find the absolute shortest distance from that point to every other node in the entire graph.

if ( d(u) + c(u,v) < d(v) )   d(v) = d(u) + c(u,v)

The Golden Translation:
"If the distance it took to get to where I am right now d(u), PLUS the cost to travel to my neighbor c(u,v), is LESS than my neighbor's current known distance d(v)... then I have found a faster route! Update my neighbor's distance."

🗺️ Part 2: Step-by-Step Graph Walkthrough

Let's look at the instructor's whiteboard process. We start by assigning a distance of 0 to our starting node, and ∞ (Infinity) to all other nodes.

Initial State: Source is 0, all others are Infinity.

The 4-Step Loop

Repeat this loop until every node is crossed out (visited):

  • 1 Select the unvisited node with the smallest known distance.
  • 2 Look at all of its immediate, unvisited neighbors.
  • 3 Apply the Golden Formula to update their distances.
  • 4 Mark the current node as "Visited" (lock it in permanently).

Walking through Step 1 (At Node 0):
Node 0 has distance 0. Neighbor Node 1 costs 4.
0 + 4 = 4. Is 4 less than ∞? Yes. Node 1's distance becomes 4.
Node 0 is now marked as Visited and locked forever.

📊 Part 3: The Table Method (Exam Standard)

When the instructor switches to a directed graph, she builds a table. This is the most organized way for a student to solve this on an exam. Every row represents a step where we select a new node.

Selected Node A B C D E
Start 0 ∞ ∞ ∞ ∞
A 0 ∞10 ∞5 ∞ ∞
C (Smallest is 5) 0 108 5 ∞14 ∞7

Notice the Overwrite! When we stood at Node C (distance 5), we looked at its neighbor B. The edge cost from C to B was 3.
5 + 3 = 8. Because 8 is less than the previously recorded 10, we crossed out the 10 and wrote 8. We found a shortcut!

Finding the Actual Path

To know the actual path (not just the distance number), the instructor says you must trace backwards. Whenever you overwrite a number with a smaller one, make a small note of which node you came from to get that shortcut.

⚠️ Part 4: The Major Drawback

Does Dijkstra work with negative edge weights?

The answer is NO.

Dijkstra is a Greedy Algorithm. Once it marks a node as "visited" (locked), it assumes it has found the absolute shortest path and refuses to ever look back at it. It naturally assumes that adding more edges will always increase your distance.

If we take path 1→3→2, the distance is 10 + (-20) = -10. But Dijkstra locks Node 2 early at distance 5 and refuses to update it!

If a negative edge exists deeper in the graph, taking a longer route might actually subtract distance. Because Dijkstra refuses to unlock nodes, it gives the wrong answer.

💡 Exam Tip: If an exam question has negative edge weights, state that Dijkstra fails and you must use the Bellman-Ford Algorithm instead.

📝 Exam Summary Checklist

  • Start: Source node distance = 0, all others = ∞.
  • Pick: Always select the unvisited node with the smallest distance.
  • Calculate: Current Node Distance + Edge Weight.
  • Compare: Is the calculation < the neighbor's current number? If yes, update it.
  • Lock: Mark the current node as visited. Cross it out on your table. Never update it again.
  • Limitation: Fails if there are negative numbers on the edges.
