1. Circular vs. Singly Linked List

The fundamental difference lies in the last node of the list.

  • In a Singly Linked List (SLL), the next pointer of the last node stores NULL (or 0).
  • In a Circular Linked List (CLL), the next pointer of the last node stores the address of the first node, creating a continuous loop.

2. The Problem with the `Head` Pointer

If we only maintain a head pointer (pointing to the first node), inserting a new node at the beginning is highly inefficient.

Why? Let's trace the logic:

  • You create a new node. You point its next to the current head.
  • You update head to point to the new node.
  • The Trap: The last node in the circle is still pointing to the old first node!

To fix this using only head, you have to create a temp pointer and traverse the entire list (O(N) time complexity) just to reach the last node and update its pointer.

The Instructor's Solution: The `Tail` Pointer

Instead of maintaining a head pointer, we only maintain a tail pointer (pointing to the last node).

If we have tail, we implicitly have head! Why? Because tail->next is ALWAYS the first node. This makes insertions at the beginning or end an O(1) operation.

3. Code: Structure & Creation

First, define the node structure. Notice we declare a global tail pointer instead of head.

struct node {
    int data;
    struct node *next;
};
struct node *tail = 0; // Initialize empty list

Now, let's write the function to create and populate the Circular Linked List.

void createCLL() {
    struct node *newnode;
    int choice = 1;
    
    while (choice == 1) { 
        // 1. Allocate dynamic memory
        newnode = (struct node*)malloc(sizeof(struct node));
        
        printf("Enter data: ");
        scanf("%d", &newnode->data);
        newnode->next = 0;
        
        // --- INSERTION LOGIC ---
        if (tail == 0) {
            // Case A: First node in the list
            tail = newnode;
            tail->next = newnode; // Points to itself!
        } else {
            // Case B: List already has nodes
            newnode->next = tail->next; // Link new node to the FIRST node
            tail->next = newnode;       // Link LAST node to the new node
            tail = newnode;             // Update tail to be the new node
        }
        
        printf("Continue? (1=yes, 0=no): ");
        scanf("%d", &choice);
    }
}

⚠️ Instructor Warning: Order of Operations

Look at the else block above. You MUST write newnode->next = tail->next; BEFORE you write tail->next = newnode;.

If you update tail->next first, you overwrite the only pointer holding the address of your first node. The circle will be broken, and you will lose the start of your list forever!

4. Traversing and Displaying the List

How do we print a circle without getting stuck in an infinite loop? In a Singly Linked List, we stopped when temp != NULL. A Circular Linked List never contains NULL.

The Wrong Way (While Loop):
If we set temp = tail->next (start at node 1) and use while (temp != tail->next), the loop will immediately fail on the first check because temp already equals tail->next!

The Right Way (Do-While Loop):
The instructor explains that a do-while loop is perfect here. It forces the code to print and move the pointer at least once before checking the condition.

void displayCLL() {
    struct node *temp;
    
    if (tail == 0) {
        printf("List is empty");
    } else {
        temp = tail->next; // Start at the FIRST node
        
        do {
            printf("%d ", temp->data); // Print current node
            temp = temp->next;         // Move to next node
        } while (temp != tail->next);  // Stop when we hit the first node again
    }
}

Visual Trace: temp starts at Node 1. The do block prints it and moves temp to Node 2. Now the while loop checks: Is Node 2 the same as Node 1? No. It keeps looping until temp wraps entirely around the circle and lands back on Node 1.
