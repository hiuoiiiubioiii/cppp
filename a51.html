Binary Tree Implementation in C

Complete Video Breakdown & Visual Notes

1. Core Concept: Visualizing the Tree & Node

A binary tree consists of nodes. Each node can have at most two children (Left and Right). Just like a Linked List uses a head pointer, a Binary Tree uses a root pointer to keep track of the starting node.

Here is how a Node sits in memory (as drawn on the whiteboard):

Address: 100
Left Ptr
(e.g., 200)
Data (4)
Right Ptr
(e.g., 300)
Instructor's Note: If a node does not have a child (a leaf node), its respective pointer will store 0 (which acts as NULL in C). During execution, we will ask the user to enter -1 to signify "no node".

2. Defining the Data Structure

Because a node contains pointers to other nodes of the exact same type, we use a "self-referential structure." We divide it into three parts: data, left pointer, and right pointer.

struct node {
    int data;                  // Stores the integer value
    struct node *left, *right; // Pointers to child nodes
};

3. The main() Function Setup

Execution starts here. We need a pointer to hold the root of our tree, and we will call a custom function named create() to build the tree and return the starting address.

void main() {
    struct node *root;   // Pointer to hold the root address
    root = create();     // Build tree, assign returned address to 'root'
}

4. Writing the Recursive create() Function

Building a tree is inherently a recursive process. You create a node, ask the function to build its left child, and then its right child.

struct node *create() {
    int x; 
    struct node *newnode;
    
    // 1. Dynamically allocate memory in the heap
    newnode = (struct node *)malloc(sizeof(struct node));
    
    printf("Enter data (-1 for no node): ");
    scanf("%d", &x);

    // 2. The Base Condition (Stops the recursion)
    if (x == -1) {
        return 0;  // 0 acts as NULL, terminating this branch
    }

    // 3. Assign Data
    newnode->data = x; 

    // 4. Recursively build Left Subtree
    printf("Enter left child of %d", x);
    newnode->left = create(); 

    // 5. Recursively build Right Subtree
    printf("Enter right child of %d", x);
    newnode->right = create();

    // 6. Return the memory address to the parent node
    return newnode; 
}

5. The Execution Trace (How the Stack Works)

"Please do not get confused... Different copies of all the local variables (x and newnode) are created for EACH function call in the stack memory."

Let's assume the user builds this tree: Root(4), Left Child(5), and no right child. Here is how the computer's stack memory executes it:

Call 1: create()
Address: 100 | x: 4
Pauses at: newnode->left = create()
Call 2: create()
Address: 200 | x: 5
Pauses at: newnode->left = create()
Call 3: create()
x: -1
Returns 0 (Triggers Base Case)
The "Aha!" Moment: When Call 3 returns `0`, the stack frame is destroyed. The computer goes back down to Call 2, and assigns that `0` to newnode(200)->left. Then Call 2 moves to the next line to check for its right child. This "dive deep, then bubble back up" action is the essence of tree recursion!
