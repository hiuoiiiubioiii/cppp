1. The Core Concept

In this technique, the array is conceptually divided into two parts:

  • Sorted Sublist: The left side. By default, the very first element (Index 0) is already considered sorted.
  • Unsorted Sublist: The right side. We take one element at a time from here and insert it into its correct position in the sorted sublist.

The Routine: Pick a value -> Store it in a variable called `temp` (leaving a hole) -> Shift sorted elements to the right if they are greater than `temp` -> Insert `temp` into the final hole.

8. Writing the Code

Here is the algorithm written exactly as explained on the whiteboard. We need an outer loop to travel left-to-right through the unsorted side, and an inner loop to travel right-to-left through the sorted side for shifting.

for (i = 1; i < n; i++) { // 1. Store the unsorted element temp = a[i]; // 2. Start comparing with the end of the sorted sublist j = i - 1; // 3. Shift elements to the right if they are greater than temp while (j >= 0 && a[j] > temp) { a[j + 1] = a[j]; // Shift right j--; // Move left } // 4. Insert temp into the correct hole a[j + 1] = temp; }

Dry Running the Loop (Inserting 10)

Let's look at the math when we were trying to insert 10 (Pass 2):

  • Outer loop: i = 2
  • temp = a[2] ➔ temp = 10
  • j = i - 1 ➔ j = 1 (a[1] is 5)
  • While loop check: Is j >= 0 (1 >= 0)? True. AND is a[1] > temp (5 > 10)? False.
  • The while loop completely skips. No shifting.
  • Insert: a[j + 1] = temp ➔ a[1 + 1] = 10 ➔ a[2] = 10. It goes right back where it started.

9. Time Complexity Analysis

Understanding the complexity is crucial for exams and interviews.

Worst-Case Scenario: O(n²)

When? The array is given in completely descending order (e.g., [6, 5, 4, 3, 2, 1]) and you want it ascending.

Why? The outer `for` loop runs `n` times. Because every new element is smaller than everything in the sorted list, the inner `while` loop must run to its maximum limit every single time to shift every element. `n` loops × `n` loops = O(n²).

Best-Case Scenario: O(n)

When? The array is already completely sorted (e.g., [1, 2, 3, 4, 5, 6]).

Why? The outer `for` loop still runs `n` times. However, the inner `while` condition a[j] > temp will fail immediately on the very first check every time. The inner loop never runs. One comparison per element = O(n).

💡 Key Student Takeaways

  • Keyword: "Shift and Insert." You do not swap elements; you shuffle larger elements to the right to make room for smaller ones.
  • Start at Index 1: Always remember the outer `for` loop starts at `i = 1`, not 0. Index 0 is your default sorted list.
  • The Safety Condition: Memorize (j >= 0 && a[j] > temp). The j >= 0 check is vital—it prevents the program from crashing by trying to read negative array indices when you reach the start of the list.
