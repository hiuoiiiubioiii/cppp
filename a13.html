Find Length of a Linked List

In-depth Video Breakdown & Study Notes

1. Visualizing the Problem

To calculate the length of a linked list, we must count the number of nodes. The instructor draws a linked list with 3 nodes on the whiteboard. Let's look at how it is structured in memory.

  • Head Pointer: Resides at address 200 and points to the first node.
  • Node 1: Data is -1, Next points to 100.
  • Node 2: Data is 1, Next points to 150.
  • Node 3: Data is 9, Next is 0 (which means NULL, marking the end).

2. The Core Concept & Golden Rule

To count the nodes, we need to traverse (walk through) the list from the first node to the last.

🚨 THE GOLDEN RULE OF LINKED LISTS:
Never use the head pointer to traverse the list! If you write head = head->next, you will lose the address of your first node forever. Your list will be lost in memory (a memory leak).

The Solution: Create a temporary pointer called temp. Point it to the start of the list (temp = head;), and move temp forward while counting.

3. The C Code Breakdown

Here is the exact code written on the whiteboard during the lecture, formatted for readability:

void getLength() {
    // 1. Initialize a counter
    int count = 0; 
    
    // 2. Create temp pointer for traversal
    struct node *temp; 
    temp = head; // Both head and temp now point to address 200
    
    // 3. Loop until temp reaches the end (0 / NULL)
    while (temp != 0) { 
        count++;               // Count the current node
        temp = temp->next;     // Move temp to the next node
    }
    
    // 4. Print final result
    printf("Length is %d", count); 
}

Instructor Note: In C, the instructor uses 0 to represent NULL. Writing temp != NULL means the exact same thing.

5. Edge Cases Addressed

Around 2:40 in the video, the instructor mentions checking for an empty list. How does our code handle it?

  • If the list is empty, head is 0 (NULL).
  • temp becomes 0 immediately.
  • The condition while(temp != 0) is evaluated as 0 != 0, which is False.
  • The loop is skipped entirely.
  • The code drops down and safely prints: Length is 0.

No special if statement is actually required because the while loop naturally handles the empty state perfectly!
