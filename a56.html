Constructing a Binary Tree

A complete visual breakdown of generating a Binary Tree using Post-Order and In-Order traversals.

The Golden Rules 🧠

Rule 1: Post-Order = (Left, Right, Root)
The root is always processed last. Therefore, if you read a Post-order sequence backwards (from Right to Left), the first number you encounter for any group is its root.


Rule 2: In-Order = (Left, Root, Right)
Once you find a root from Rule 1, locate it in the In-order sequence. Everything to its left belongs in the Left Subtree, and everything to its right belongs in the Right Subtree.

Given Data:

Post: 9, 1, 2, 12, 7, 5, 3, 11, 4, 8
In: 9, 5, 1, 7, 2, 12, 8, 4, 3, 11

Step 1: Find the Main Root

Instructor's Logic You cannot find the root using In-order because the root is somewhere in the middle. You must use Post-order. The very last element is the main root of the entire tree.
Action Scan Post-order from Right to Left. The last element is 8.

Final Step: Verify Your Work ✅

The instructor provides a highly recommended pro-tip for exams:

Ignore the arrays written on the board. Look ONLY at the drawing of the tree you just created. Manually calculate the Post-order and In-order traversals based on your drawing.

If your manual calculation matches the sequences given in the original problem exactly, your tree is 100% correct.
