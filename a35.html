Infix to Prefix Conversion

Welcome to your study feed! Here, we break down exactly how to convert mathematical expressions from Infix (operators between operands) to Prefix (operators before operands). Keep scrolling to learn the prerequisites, manual conversion, and the Stack Algorithm.

1. Prerequisites 🔑

Before we start, you must memorize operator Precedence and Associativity. This dictates which operator is solved first.

Operator Precedence Associativity
^ Highest (3) Right to Left (R → L)
* , / Medium (2) Left to Right (L → R)
+ , - Lowest (1) Left to Right (L → R)

2. The Basics (Manual Conversion)

Let's manually convert a simple Infix expression to Prefix: A + B * C

Step 1: A + B * C
/* * has higher precedence than +. Prefix means move operator BEFORE operands */

Step 2: A + (*BC)
/* Treat (*BC) as one operand. Move + to the front. */

Final: + A * B C
Why do we need a Stack Algorithm?
Doing this manually requires scanning the expression multiple times. For a computer, this is inefficient. A Stack allows the computer to convert complex expressions by scanning the string only once!

3. The Stack Algorithm Rules 📜

To use a Stack for Prefix conversion, follow these exact steps carefully:

  • Reverse the Infix Expression. Read from right to left.
    Instructor Trick: Do NOT swap parentheses when reversing! Leave ) as ) and ( as (. This means ) will act as our "opening" trigger to push, and ( will trigger a pop.
  • Scan the reversed string from Left to Right:
    • 👉 If Operand (A-Z): Print directly to output.
    • 👉 If ) : Push to Stack unconditionally.
    • 👉 If ( : Pop from Stack and print to output until you find ). Discard both brackets.
    • 👉 If Operator:
      1. If Stack is empty or top is ) ➜ Push.
      2. If incoming is HIGHER precedence than Stack top ➜ Push.
      3. If incoming is LOWER precedence ➜ Pop and print top of stack until incoming is higher/equal, then push.
      4. If incoming is EQUAL precedence ➜ Check Associativity! Because we reversed the string, the rule flips. For Left-to-Right (+,-,*,/), you PUSH it!
  • Reverse the final printed string to get your Prefix answer!

5. Manual Verification (Without Stack) ✍️

To prove the Stack algorithm works perfectly, the instructor solves it manually using standard math rules (BODMAS/PEMDAS).

Original: K + L - M * N + (O ^ P) * W / U / V * T + Q

1. Parentheses first: (O ^ P)
K + L - M * N + ^OP * W / U / V * T + Q

2. Mult/Div (L to R). First is M * N
K + L - *MN + ^OP * W / U / V * T + Q

3. Next Mult/Div: ^OP * W
K + L - *MN + *^OPW / U / V * T + Q

4. Next Mult/Div: *^OPW / U
K + L - *MN + /*^OPWU / V * T + Q

5. Next Mult/Div: /*^OPWU / V
K + L - *MN + //*^OPWUV * T + Q

6. Next Mult/Div: //*^OPWUV * T
