1. The Initial Setup & Memory

Before we can delete an element, we must have an array created and populated with data. Let's assume the user created an array of max capacity 50, but only inputted 5 elements.

Crucial Rule: "Position" is how humans count (1st, 2nd, 3rd). "Index" is how the computer counts (0, 1, 2).

Formula: Index = Position - 1

Here is our initial array in memory. Notice how the memory addresses increment by 4 bytes (assuming standard integer size).

2. The Core Problem: The Gap

The instructor asks: "I want to delete the data from Position 2."

Position 2 means Index 1. The value is 2. Because arrays use contiguous memory, we cannot just delete '2' and leave a blank hole in the middle of our memory block.

The Solution: Shifting. We must shift all elements on the right side one step to the left, overwriting the deleted element.

3. The C Code Implementation

Here is exactly how the instructor translates the shifting logic into C code on the whiteboard.

// 1. Get user input for position
int pos;
printf("From which position you want to delete: ");
scanf("%d", &pos);

// 2. Bound Checking (Invalid inputs)
if (pos <= 0 || pos > size) {
    printf("Invalid position");
}
else {
    // Optional: Store deleted item
    int item = a[pos - 1]; 

    // 3. The Shifting Loop
    for (i = pos - 1; i < size - 1; i++) {
        a[i] = a[i + 1];
    }
    
    // 4. Reduce total size
    size--; 
}
Instructor Note: Why loop to size - 1?
If we loop all the way to size, the code will try to execute a[4] = a[5]. Since a[5] is outside our populated data, it pulls in garbage data! Stopping at size - 1 prevents this underflow/garbage issue.

4. PRO-TIP: Unsorted Array Trick (O(1) Deletion)

At the end of the video, the instructor reveals a massive optimization trick frequently asked in interviews.

If the array is unsorted (order doesn't matter), you do not need to use a loop to shift elements! Shifting is slow. Instead, just copy the very last element in the array, paste it over the element you want to delete, and decrease the size.

// O(1) Trick - No loops required!
a[pos - 1] = a[size - 1];
size--;

5. Time Complexity Summary

Make sure to memorize these scenarios for your exams:

Scenario Time Complexity Explanation
Beginning of Array O(N) Worst case. You must shift every single element to the left.
Middle of Array O(N) Average case. You have to shift roughly half the elements.
End of Array O(1) Best case. No shifting required, just size--.
Unsorted Optimization O(1) Using the swap-with-last-element trick detailed above.
