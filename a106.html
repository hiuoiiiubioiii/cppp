1. Core Concept: Not a Comparison Sort

Unlike Bubble Sort, Quick Sort, or Merge Sort, Counting Sort is not a comparison sort. We never check if a[i] > a[j].

"Sorting acc to keys by counting the elements having distinct key values."

Because we bypass comparisons, we can beat the theoretical limit of O(n log n) and achieve a time complexity of O(n) Linear Time under specific conditions.

Setup

2. The Example Array

Let's look at the exact array the instructor wrote on the whiteboard to trace the algorithm step-by-step.

Total Elements (n)

n = 17

Maximum Value (k)

k = 9

* Notice that the elements range from 0 to 9. Therefore, our target range is [0, k].

3. Step 1 & 2: Creating the Count Array

Step 1: We create a new array named count. Because our max value is k=9, the array needs to have indices from 0 to 9. Therefore, its size is k + 1 = 10. We initialize all slots to 0.

Step 2: We scan the original array from left to right. Every time we see a value, we treat that value as an index in our count array and increment it by 1.
Example: See a '2'? Go to index 2 in count array and add 1.

Notice: index 2 holds '4' because the number 2 appears four times in the original array.

4. Step 3: The Cumulative Count

We don't just want to know *how many* times a number appears. We need to know where to place it in the final sorted array. To do this, we add the current count to the previous count.

count[i] = count[i] + count[i-1]

What does this new array tell us?

Look at index 2. The value is 10. This means that all the 0s, 1s, and 2s take up exactly 10 slots in total. Therefore, the very last '2' belongs at position 10 (which is index 9 in a 0-based array).

5. Step 4: Building the Output Array (Stability)

Now we create a final array b of size n to hold our sorted elements.

Crucial Rule: We must scan the original array backwards (from right to left, i.e., index 16 down to 0). Why? To maintain Stability! This ensures that if there are two '9's, the one that appeared first originally stays first in the sorted array.

The Loop Mechanics:

  1. Look at element in original array (e.g., let's say it's 9).
  2. Go to index 9 in the count array. The value there is 17.
  3. Decrement the value by 1 (it becomes 16). We do this because array indices are 0-based (0 to 16).
  4. Place the number 9 at index 16 in output array b.

* After building array 'b', Step 5 is simply copying everything from 'b' back into original array 'a'.

6. Complete C Code Implementation

Here is the full function exactly as the instructor outlined it on the board:

void countSort(int a[], int n, int k) {
    int count[k+1]; // Size depends on max value 'k'
    int b[n];       // Output array, size depends on 'n'
    int i;
    
    // Step 1: Initialize count array to 0
    for(i = 0; i <= k; i++) 
        count[i] = 0;

    // Step 2: Store the frequency of each element
    for(i = 0; i < n; i++) 
        ++count[a[i]];

    // Step 3: Update count array to store cumulative sum
    for(i = 1; i <= k; i++) 
        count[i] = count[i] + count[i-1];

    // Step 4: Build the output array (Scan backwards for stability)
    for(i = n - 1; i >= 0; i--) {
        b[--count[a[i]]] = a[i]; // Pre-decrement handles 0-based index
    }

    // Step 5: Copy the sorted elements into original array
    for(i = 0; i < n; i++) 
        a[i] = b[i];
}

7. Time Complexity Analysis

Let's look at the function and add up the execution time of every for loop:

  • 1 Loop 1 (Init): Runs from 0 to k → O(k)
  • 2 Loop 2 (Freq): Runs from 0 to n → O(n)
  • 3 Loop 3 (Sum): Runs from 1 to k → O(k)
  • 4 Loop 4 (Build b): Runs from n to 0 → O(n)
  • 5 Loop 5 (Copy): Runs from 0 to n → O(n)

Total Time = O(k) + O(n) + O(k) + O(n) + O(n) = O(3n + 2k)

Dropping the constants yields our final complexity:

O(n + k)

Space Complexity: We created count (size k) and b (size n). Therefore, auxiliary space is also O(n + k).

Warning

8. Drawbacks & Limitations

1. Memory Waste on Huge Ranges

If array a = {10000}, $n=1$, but $k=10000$. We are forced to create a count array of size 10,001 just to sort a single element! If $k$ becomes proportional to $n^2$, time complexity degrades to O(n²). Counting sort is only useful when $k$ is roughly proportional to $n$.

2. No Negative Numbers (Naturally)

We use elements as array indices. You cannot query count[-5].

Workaround: You have to find the minimum negative number, take its absolute value, and add it to every element to make them positive, then sort, and subtract it back later.

3. No Floating Point Numbers

Array indices must be integers. You cannot query count[1.5]. Counting sort strictly requires non-negative integers representing distinct keys.

Instructor's Pro Tip: If elements are scattered across a massive random range (e.g., 652, 0, 999, 10000), do not use Counting Sort alone. Instead, use Radix Sort, which uses Counting Sort as a subroutine digit-by-digit!
