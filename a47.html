Module: Deque (Double Ended Queue)

1. Standard Queue vs. Deque

To understand a Deque (pronounced "deck"), you must first understand the limitations of a standard Queue.

The Standard Queue
← Delete
← Insert
Front
Rear

Rule: Insertion can only happen at the rear. Deletion can only happen at the front.

The Deque (Double Ended Queue)
→ Insert ← Delete
Delete → Insert ←
Front
Rear

Definition: A Deque is a generalized version of a queue. It is a linear list in which insertion (enqueue) and deletion (dequeue) operations are allowed from both ends.

2. Properties of a Deque

Because you can insert and delete from both ends, a Deque is highly versatile. It supports the properties of both major linear data structures.

What she writes on the board:
Stack - LIFO (Last In, First Out)
Queue - FIFO (First In, First Out)
  • How to use as a Stack: Limit all insertions and deletions to just one end of the Deque. This perfectly mimics Last-In-First-Out behavior.
  • How to use as a Queue: Restrict insertion to one end and deletion to the opposite end. This mimics standard First-In-First-Out behavior.

3. Types of Deques

There are two primary variations of a Deque based on applying specific constraints to either the input or the output.

Type A: Input-Restricted Deque

We put a restriction on the input (insertion). Insertion is allowed from only one end, but deletion is allowed from both ends.

Type B: Output-Restricted Deque

We put a restriction on the output (deletion). Deletion is allowed from only one end, but insertion is allowed from both ends.

4. Operations on a Deque

Here is the list of standard operations you can perform on a Deque. (Actual code implementation is deferred to the next video lecture).

  • insertAtFront()
  • deleteFromFront()
  • insertAtRear()
  • deleteFromRear()
  • getFront() // Peek: view front element without removing
  • getRear() // Peek: view rear element without removing
  • isFull() // Returns true if no more space
  • isEmpty() // Returns true if 0 elements

5. Memory Representation

How do we actually build this in memory? You can use a Doubly Linked List, but the most common array-based method is the Circular Array.

The "False Full" Problem (Linear Array)

4
2

R

If `rear` is at the last index, you can't insert a new element at the rear, even if index 0 is empty! The system will incorrectly say the Queue is Full.

The Circular Solution

Logically connecting the end back to the beginning allows the rear pointer to wrap around to index 0. This achieves O(1) time complexity for all operations.

6. Real World Applications

Because a Deque acts as both a stack and a queue, it inherits their applications (like Undo/Redo in editors), plus some unique ones.

A. Palindrome Checker

→ r a d a r ←

Remove characters from both the front and rear simultaneously. If they always match, it's a palindrome.

B. Multiprocessor Scheduling (Work Stealing)

Also noted on the whiteboard by the instructor as the "A-steel Algorithm". This handles load balancing between multiple computer processors.

P1
F
R
P2
F
R
  • Each processor has its own Deque of threads (jobs). Normally, a processor takes jobs from the front of its own Deque.
  • If P1 finishes all its jobs (its deque is empty), it won't sit idle.
  • P1 will "steal" a thread from the rear of P2's Deque, while P2 continues processing from its own front.
  • P1 adds the stolen thread to its own front and executes it. This is why a double-ended queue is mandatory!
