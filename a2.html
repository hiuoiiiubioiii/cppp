  • Insertion (Covered today)

Code: Populating & Traversing

C arrays do not have built-in "bounds checking" at runtime. We must declare a large maximum size at compile time, but ask the user for the actual size they want to use at runtime.

int a[50], size, i;

// 1. Get the actual size from user
printf("Enter size of array: ");
scanf("%d", &size); 

// 2. Populate the array
printf("Enter elements of array: ");
for(i = 0; i < size; i++) {
    // '&' is required to store data at the memory address!
    scanf("%d", &a[i]);  
}

// 3. Traverse and Print
printf("Elements in array are: ");
for(i = 0; i < size; i++) {
    // NO '&' here. We just want the value, not the address.
    printf("%d ", a[i]); 
}

3. Array Insertion (Specific Position)

Inserting data at a specific position is the most complex insertion operation. Once you master this, inserting at the beginning or end is trivial.

The "Overflow" Condition
Before inserting, you must write code to check if the array is already full. If we declared int a[50] and size == 50, there is no room left. Trying to insert will cause an Array Overflow.

Position vs. Index

Humans count starting from 1 (Position). Computers count starting from 0 (Index).
Formula: Target Index = Position - 1
Example: If a user wants to insert at Position 3, the target index is 2.

The Shifting Concept

Assume we have 5 elements. We want to insert the number 10 at Position 3 (Index 2). You CANNOT just write a[2] = 10, because you will overwrite and permanently lose the existing data. You must shift elements to the right first!

Golden Rule: You must always start shifting from the end of the array moving left. If you shift left-to-right, you overwrite your own data.

Complete C Code for Insertion

int num, pos;

printf("Enter data you want to insert: ");
scanf("%d", &num);

printf("Enter Position: ");
scanf("%d", &pos);

// Bounds & Overflow Checking
if(pos <= 0 || pos > size + 1) {
    printf("Invalid Position");
}
else if(size >= 50) {
    printf("Array Overflow");
}
else {
    // 1. Shift elements to the right
    // Start at the last element (size-1), stop at target index (pos-1)
    for(i = size - 1; i >= pos - 1; i--) {
        a[i + 1] = a[i];  
    }
    
    // 2. Insert the new number
    a[pos - 1] = num;
    
    // 3. Update the size of the array
    size++;
}

4. Modifying Insertion for Other Scenarios

Inserting at the Beginning

If you specifically want an algorithm just for inserting at the very beginning (Index 0), modify the loop so it always stops at index 0:

// Shift everything from the end down to index 0
for(i = size - 1; i >= 0; i--) {
    a[i + 1] = a[i];
}
a[0] = num;
size++;

Inserting at the End

If you only want to insert at the end, no shifting is required! The current size integer naturally acts as the index for the first empty slot.

a[size] = num; 
size++;

5. Time Complexity & Pro-Tip

If the array is sorted, or if the relative order of the elements must be maintained, you are forced to shift elements. This determines the time complexity:

  • Best Case: $\Theta(1)$ (Inserting at the very end, no shifting required).
  • Worst Case: $\Theta(n)$ (Inserting at the very beginning, must shift every single element).
  • Average Case: $\Theta(n - p)$ where $p$ is the position (You only shift the elements that come after the chosen position).

💡 Pro-Tip: The "Unsorted Array" Trick

If the array is unsorted and the order of elements does not matter, you can achieve $\Theta(1)$ time complexity for inserting at any position, completely avoiding the shifting loop!

Algorithm: Instead of shifting everything, just take the element currently sitting at the target index, throw it to the very end of the array, and put your new number in the newly vacated spot.

// 1. Move element at target index to the end
a[size] = a[pos - 1]; 

// 2. Put the new number in the target index
a[pos - 1] = num;

// 3. Increment size
size++;

(This is a highly useful trick for competitive programming and technical interviews!)
