Part 1: Prefix

Understanding Prefix Notation

In prefix notation, operators are written before their operands.

<operator> <operand 1> <operand 2>

Substituted Prefix Expression

- + 2 * 3 4 / 16 ^ 2 3

Prefix: Manual Evaluation

The Rule:

Scan from Right to Left. Find the first <op> <val> <val> pattern, evaluate it, rewrite, and repeat.

Step 1 - + 2 * 3 4 / 16 ^ 2 3 2^3 = 8
Step 2 - + 2 * 3 4 / 16 8 16/8 = 2
Step 3 - + 2 * 3 4 2 3*4 = 12
Step 4 - + 2 12 2 2+12 = 14
Step 5 - 14 2 Result = 12

Instructor Note: Manual evaluation requires multiple passes through the expression, which is time-consuming for computers.

Prefix: Stack Evaluation (Programmatic)

Stack Rules for Prefix:

  1. Scan Right to Left.
  2. If Operand ➡️ Push to stack.
  3. If Operator ➡️ Pop 2 elements.
    Op1 = Top Element
    Op2 = Next Element
  4. Calculate Op1 [operator] Op2 and push result.
Token (R to L) Action Stack (Bottom → Top)
3Push 3[ 3 ]
2Push 2[ 3, 2 ]
^Pop 2, Pop 3.
Calc: 2 ^ 3 = 8. Push 8		[ 8 ]
16Push 16[ 8, 16 ]
/Pop 16, Pop 8.
Calc: 16 / 8 = 2. Push 2		[ 2 ]
4Push 4[ 2, 4 ]
3Push 3[ 2, 4, 3 ]
*Pop 3, Pop 4.
Calc: 3 * 4 = 12. Push 12		[ 2, 12 ]

+Pop 2, Pop 12.
Calc: 2 + 12 = 14. Push 14		[ 2, 14 ]
-Pop 14, Pop 2.
Calc: 14 - 2 = 12. Push 12		[ 12 ]

Return stack[top] ➡️ 12

Whiteboard Algorithm: Prefix

Scan Prefix expr. from right to left
for each char in prefix expr.
do
    If operand is there, push it onto stack
    else if operator is there, pop 2 elements
        op1 = top element
        op2 = next to top element
        result = op1 operator op2
        
    Push result onto stack
return stack[top]
Part 2: Postfix

Understanding Postfix Notation

In postfix notation, operators are written after their operands.

<operand 1> <operand 2> <operator>

Substituted Postfix Expression

2 3 4 * + 16 2 3 ^ / -

Postfix: Manual Evaluation

The Rule:

Scan from Left to Right. Find the first <val> <val> <op> pattern, evaluate it, rewrite, and repeat.

Step 1 2 3 4 * + 16 2 3 ^ / - 3*4 = 12
Step 2 2 12 + 16 2 3 ^ / - 2+12 = 14
Step 3 14 16 2 3 ^ / - 2^3 = 8
Step 4 14 16 8 / - 16/8 = 2
Step 5 14 2 - Result = 12

Postfix: Stack Evaluation (Programmatic)

CRITICAL DIFFERENCE: POP ORDER

When you pop two elements for Postfix, the order is reversed from Prefix. Because you scanned left to right, the most recent number pushed is the second operand.

Op2 = Top Element
Op1 = Next Element
Token (L to R) Action Stack (Bottom → Top)
2Push 2[ 2 ]
3Push 3[ 2, 3 ]
4Push 4[ 2, 3, 4 ]
*Pop Op2: 4, Pop Op1: 3.
Calc: 3 * 4 = 12. Push 12		[ 2, 12 ]
+Pop Op2: 12, Pop Op1: 2.
Calc: 2 + 12 = 14. Push 14		[ 14 ]
16Push 16[ 14, 16 ]
2Push 2[ 14, 16, 2 ]
3Push 3[ 14, 16, 2, 3 ]
^Pop Op2: 3, Pop Op1: 2.
Calc: 2 ^ 3 = 8. Push 8		[ 14, 16, 8 ]
/Pop Op2: 8, Pop Op1: 16.
Calc: 16 / 8 = 2. Push 2		[ 14, 2 ]
-Pop Op2: 2, Pop Op1: 14.
Calc: 14 - 2 = 12. Push 12		[ 12 ]

Final Stack State

12

Return stack[top] ➡️ 12

Summary

The Two Golden Rules to Memorize

Prefix

Scan Direction

Right ⬅️ Left

Stack Pop Order

1st Pop = Op1
2nd Pop = Op2

Postfix

Scan Direction

Left ➡️ Right

Stack Pop Order

1st Pop = Op2
2nd Pop = Op1
