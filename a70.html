🌳 Deletion in Splay Trees

Top-Down Approach Breakdown

Unlike standard Binary Search Trees where you swap a node with its inorder successor or predecessor, in a Top-Down Splay Tree, you NEVER delete an internal node directly.

📌 The 4-Step Golden Rule:

  1. Search for the element to be deleted.
  2. Splay that element to the root of the tree.
  3. Delete the root node. This splits the tree into two disjoint subtrees: a Left Subtree (L) and a Right Subtree (R).
  4. Join the two subtrees back together.
🔗 How the "Join" Operation Works:
1. Go to the Left Subtree (L).
2. Find the maximum element in L.
3. Splay that maximum element so it becomes the new root of L. (Since it's the max element, its right-child pointer is guaranteed to be NULL!)
4. Attach the Right Subtree (R) as the right child of this new root.

🚨 Audio Notes: Edge Cases Explained

The instructor verbalizes two important edge cases you must account for in your code:

  • What if you try to delete a node that isn't in the tree? (e.g., Delete 20)
    You search for 20 and eventually hit a NULL. According to Splay rules, you must splay the last accessed non-null node (e.g., 17) to the root. Once 17 is at the root, you compare the root data to the target data. Since 17 != 20, you simply print "Key not present" and stop.

  • What if a subtree is empty?
    If L is empty (like in Example 3), R becomes the whole tree. If R is empty, L simply becomes the whole tree.

💻 Algorithm / Pseudo-code

Here is the code structure the instructor mapped out on the whiteboard:

// 1. Handle an empty tree
if (root == NULL) 
    return NULL;

// 2. Splay the target data to the root
Splay(root, data);

// 3. Check if the data actually existed in the tree
if (data != root->data) {
    print "Key not present";
    return root; 
}

// 4. At this point, the root IS the node we want to delete.
// Hold onto the current root so we can free it later.
temp = root;

// 5. Edge Case: What if there is no Left Subtree?
if (root->left == NULL) {
    root = root->right;   // Right Subtree becomes the tree
} 
else {
    // 6. Standard Join Operation
    // Note: Calling Splay with the target 'data' on the left subtree 
    // naturally splays the maximum element to the top, because 'data'  
    // is greater than everything in the left subtree.
    Splay(root->left, data);
    
    // The new root is now the max element of the left subtree.
    root = root->left; 
    
    // Attach the old root's right subtree.
    root->right = temp->right; 
}

// 7. Delete the old root from memory
free(temp);

// 8. Return the new tree
return root;
