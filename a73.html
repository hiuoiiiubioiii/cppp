Creating a B-Tree of Order 5

Complete Visual Study Guide & Insertion Walkthrough (Values 1 to 20)

📌 The Instructor's Golden Rules

Before we insert anything, we must calculate the specific constraints of our B-Tree based on its order (m).

Order (m) m = 5
Max Children per Node m = 5
Max Keys per Node m - 1 = 4
  • Rule 1: Keys inside a node must always be in sorted (ascending) order.
  • Rule 2: New elements are ALWAYS inserted into a leaf node first. You never insert directly into an internal/parent node.
  • Rule 3: If a node gets 5 keys, it overflows. You must find the median, push it up to the parent, and split the remaining keys into two children.

Phase 2: The First Split

Action: Insert 5.

We attempt to insert 5, but the node is full! We imagine the array [1, 2, 3, 4, 5]. The median (middle element) is calculated as (4+1)/2 = 2.5 (we round up to the 3rd element). The median is 3.

The Split: 3 moves UP to become the new root. Elements less than 3 [1, 2] become the left child, and elements greater [4, 5] become the right child.

The median is 6. 6 moves UP to the parent node. The parent node currently just has [3], so it happily accepts 6, becoming [3 | 6]. The leaf splits.

Phase 7: The Double Split (Increasing Tree Height)

⚠️ Pay close attention here, this is where most mistakes happen!

Action: Insert 15, 16, then 17.

15 and 16 fill the rightmost leaf. Inserting 17 causes overflow: [13, 14, 15, 16, 17]. The median 15 wants to move up.

Wait! The parent root [3, 6, 9, 12] is already full! We cannot just push 15 in. We must split the root.

Imagine the root receiving 15: [3, 6, 9, 12, 15]. The median of this group is 9. So, 9 moves UP to become a brand new root node. The tree height increases!

✅ Quick Summary Checklist for Exams

  • Check Max Keys: Always keep track of m - 1 (in this case, 4). Stop and split the moment you hit 5 elements.
  • Insert at Leaf: Never jam a number into a parent node directly from the input sequence. Trace the path down to the correct leaf first.
  • Find the Median: When a leaf overflows (reaches 5 items), list them in sorted order. The 3rd item is always the median.
  • Push Up & Split: The median goes up. The two items smaller stay in the left node. The two items larger go to a new right node.
  • Domino Effect: If pushing a median up causes the parent to have 5 items, you must repeat the exact same split process on the parent.
