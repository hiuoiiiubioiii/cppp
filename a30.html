Implementation of Stack using Linked List

Continuous Scroll Study Notes • Visualizer • C Code Definitions

1. Core Concepts & The "Why"

Before writing code, we must understand the fundamental rules of the Stack and the Linked List data structures.

The Prerequisites

Stack: Follows the LIFO (Last In, First Out) principle. All insertions (push) and deletions (pop) happen at one specific end, called the Top.

Singly Linked List: A collection of dynamic nodes. Each node contains a data part and a link part (a pointer to the next node's address).

The Golden Rule of Stacks:
The time complexity for both push and pop operations MUST be O(1) (Constant Time). This dictates how we implement the stack.

Where do we Insert and Delete?

We have a Linked List starting with a Head pointer. We have two choices for where the stack operations can take place:

Option A: At the Tail (End)

To push or pop a node at the very end of a linked list, you must traverse the entire list from Head to Tail.

Time Complexity: O(n)

❌ Rejected

Option B: At the Head (Beginning)

To push or pop a node at the beginning, you simply update the Head pointer. No traversal required!

Time Complexity: O(1)

✅ Accepted

Conclusion: To match stack terminology, we will rename the Head pointer to Top. Every new element will be inserted at the Top, and every element removed will be taken from the Top.

Notice the Mechanics:
  • When pushing, the new node's link stores the current address of Top.
  • Then, Top updates to point to the new node.
  • When popping, Top drops down to the next node (Top = Top->link), and the old node is freed from memory.

3. C Code Implementation

Unlike arrays, implementing a stack with a linked list relies on dynamic memory allocation (using malloc and free). This means there is no fixed maximum size—no traditional "Overflow" condition!

1. Structure Definition & Global Setup

#include <stdio.h>
#include <stdlib.h>

// Define the Node structure
struct node {
    int data;
    struct node *link; // Pointer to the next node
};

// Declare Top globally, initialized to 0 (NULL)
struct node *top = 0;

2. The Push Function O(1)

void push(int x) {
    struct node *newnode;
    
    // Dynamically allocate memory for the new node
    newnode = (struct node*)malloc(sizeof(struct node));
    
    // Store the data
    newnode->data = x;
    
    // New node points to whatever Top is currently pointing to
    newnode->link = top; 
    
    // Update Top to point to our newly inserted node
    top = newnode;
}

3. The Pop Function O(1)

void pop() {
    struct node *temp;
    temp = top; // Temp holds the node to be deleted
    
    // Check for Underflow (Empty Stack)
    if(top == 0) {
        printf("Underflow Condition! Stack is empty.\n");
    } else {
        printf("Popped element is %d\n", top->data);
        
        // Drop the Top pointer down to the next node
        top = top->link;
        
        // Release memory back to the heap to prevent memory leaks
        free(temp); 
    }
}

4. Display & Peek Functions

void peek() {
    if(top == 0) {
        printf("Stack is empty\n");
    } else {
        printf("Top element is %d\n", top->data);
    }
}

void display() {
    struct node *temp;
    temp = top;
    
    if(top == 0) {
        printf("Stack is empty\n");
    } else {
        // Traverse until the end (NULL) is reached
        while(temp != 0) {
            printf("%d ", temp->data);
            temp = temp->link; // Move pointer forward
        }
    }
}

4. Final Summary for Exam Prep

  • Time Complexities: Push = O(1), Pop = O(1), Peek = O(1), Display = O(n).
  • Memory Allocation: Dynamic. Memory is requested at runtime using malloc(). No pre-defined size array is needed.
  • Insertion/Deletion Rule: Always insert at the head (Top), always delete from the head (Top).
  • Memory Leaks: Always use a temporary pointer and free(temp) when popping an element to release the heap memory.
