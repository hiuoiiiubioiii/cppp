Study Notes: Reversing a Circular Linked List

Scroll down to explore the concepts, visualizations, and code breakdown step-by-step.

1. The Core Concept

What does "Reversing" mean in the context of a Linked List?

  • Do not swap the data. We only reverse the links (the pointers) between the nodes.
  • In a Circular Linked List (CLL), the last node points back to the first node. We usually only maintain a `tail` pointer pointing to the last node.
  • The Goal: Make node 2 point to node 1, node 3 to node 2, etc. Finally, update the `tail` pointer so it points to the original first node (which becomes the new last node).

3. The Strategy & Three Pointers

If we just break a link to reverse it, we immediately lose the address of the rest of the list. To solve this, the instructor introduces three temporary pointers:

prev
current
nextnode

These will slide across the list together. nextnode jumps ahead to hold the rest of the list, allowing current to safely turn around and point to prev.

Part B: The Main Reversal Loop

We traverse until current reaches the tail. Look closely at the order of operations inside the loop!

        while(current != tail) {
            /* Step 1: prev catches up to current */
            prev = current;             
            
            /* Step 2: current moves forward */
            current = nextnode;         
            
            /* Step 3: nextnode jumps ahead to save the remaining list */
            nextnode = current->next;   
            
            /* Step 4: Break the link and reverse it! */
            current->next = prev;       
        }

Tracing Iteration 1:

  • prev gets 500.
  • current moves to 200.
  • nextnode secures 150.
  • current->next (Node 200's pointer) is changed to 500. Link is reversed!

Note: This loop stops when current == 600 (the tail). It breaks before reversing the tail node's link.

Part C: Finalizing Outside the Loop

Because the loop broke early, we have manual clean-up to do. We must reverse the final node, close the circular loop, and officially move the tail.

        // After loop: current is 600, prev is 150, nextnode is 500

        /* 1. Make the old First Node point to the old Last Node */
        nextnode->next = tail;   // 500's next becomes 600
        
        /* 2. Update the global tail pointer */
        tail = nextnode;         // tail becomes 500
    }
}
