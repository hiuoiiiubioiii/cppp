Circular Queue in C

Complete In-Depth Breakdown | Using Arrays

🔄 1. The Problem with Linear Queues

Before writing code, the instructor outlines exactly why we need Circular Queues. In standard Linear Queues implemented with arrays, we face a major memory waste issue.

  • FIFO Rule: Queues operate on First-In-First-Out. Insertions happen at the rear, deletions happen at the front.
  • The Drawback: When rear reaches the last index of the array (N-1), the queue is declared full. However, if we have dequeued elements, indexes 0, 1, etc., are actually empty! In a linear queue, we cannot reuse this space.
  • The Solution: Logically connect the last index back to index 0. If the rear reaches the end, it wraps around to the beginning, creating a Circular Queue.
The Magic Formula: To achieve the wrap-around effect, never use rear++ or front++. Instead, use modulo arithmetic:
Next Position = (Current Position + 1) % N

🛠 2. Global Setup & Variables

The instructor sets up the array and pointers globally so all functions can access them. Initially, both pointers are set to -1 to represent an empty queue.

// Define max size of the Queue
#define N 5  

// Create the array and pointers
int queue[N];
int front = -1;
int rear = -1;

📥 3. The Enqueue Operation

When inserting data (x) into the queue, we must check three specific conditions. The instructor explains them as follows:

Condition 1: Is the Queue Completely Empty?

If both pointers are -1, it means we are inserting the very first element. We must move both pointers to 0.

Condition 2: Is the Queue Full? (Overflow)

Because it's circular, the queue is full if the next position of rear hits the front pointer.

Check: if ((rear + 1) % N == front)

Condition 3: Normal Insertion (Wrap-Around)

If it's not empty and not full, we increment rear using the modulo formula and insert the data.

void enqueue(int x) {
    // Case 1: Queue is empty
    if (front == -1 && rear == -1) {
        front = 0;
        rear = 0;
        queue[rear] = x;
    } 
    // Case 2: Queue is full
    else if ((rear + 1) % N == front) {
        printf("Queue is full\n");
    } 
    // Case 3: Normal insertion / wrap-around
    else {
        rear = (rear + 1) % N;
        queue[rear] = x;
    }
}

📤 4. The Dequeue Operation

Deleting data happens at the front pointer. Similar to insertion, there are three distinct conditions.

Condition 1: Is the Queue Empty? (Underflow)

Check: if (front == -1 && rear == -1). If true, there is nothing to delete.

Condition 2: Only One Element Left

If front == rear, both pointers are pointing to the exact same index. This means only one item exists. After deleting it, the queue becomes empty again, so we must reset front and rear back to -1.

Condition 3: Normal Deletion

Print the element at the front, then move front to the next logical position using the modulo formula.

void dequeue() {
    // Case 1: Queue is empty
    if (front == -1 && rear == -1) {
        printf("Queue is empty\n");
    } 
    // Case 2: Only one element exists
    else if (front == rear) {
        printf("Dequeued %d\n", queue[front]);
        front = -1;
        rear = -1;  // Reset the queue
    } 
    // Case 3: Normal deletion / wrap-around
    else {
        printf("Dequeued %d\n", queue[front]);
        front = (front + 1) % N;
    }
}

👁️ 5. The Display Operation

The instructor highlights a critical difference here. In a linear queue, a simple for (i=front; i<=rear; i++) loop works. In a circular queue, rear might be numerically smaller than front (e.g., front is 3, rear wrapped around to 1). A standard for loop will fail.

The Strategy:

  1. Check if empty.
  2. Start a temporary variable i at front.
  3. Use a while (i != rear) loop. Inside, print the element and increment circularly: i = (i + 1) % N.
  4. Crucial Step: The loop breaks the moment i hits rear. This means the last element is skipped! We must manually print queue[rear] outside the loop.
void display() {
    int i = front; 
    
    if (front == -1 && rear == -1) {
        printf("Queue is empty\n");
    } 
    else {
        printf("Queue is: ");
        // Loop until i reaches rear
        while (i != rear) {
            printf("%d ", queue[i]);
            i = (i + 1) % N; // Move circularly
        }
        // Print the final element at rear
        printf("%d\n", queue[rear]);
    }
}

🫣 6. The Peek Operation

The peek() function is incredibly simple. It operates exactly like a linear queue. It just looks at the front of the queue and prints it without removing it.

void peek() {
    if (front == -1 && rear == -1) {
        printf("Queue is empty\n");
    } 
    else {
        printf("Front element is %d\n", queue[front]);
    }
}
