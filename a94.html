All Pair Shortest Path (Floyd-Warshall)

1. The Core Concept

The All Pair Shortest Path problem asks us to find the shortest distance from every single node to every other node in a graph.

Why not use Dijkstra's Algorithm?
1. Dijkstra is a "Single Source" algorithm. You'd have to run it \(V\) times.
2. Crucial Flaw: Dijkstra fails if the graph has negative edge weights. Floyd-Warshall handles negative weights perfectly!

The Strategy: Dynamic Programming
We break the complex problem down. To find the shortest path from Node \(i\) to Node \(j\), we ask: "Is it shorter to go directly from \(i\) to \(j\), or should we go through an intermediate 'via' node \(k\)?"

8. The Core Formula & Logic

The math used to generate these matrices is expressed by this Dynamic Programming formula:

\[ D^k[i, j] = \min \left( D^{k-1}[i, j], \; D^{k-1}[i, k] + D^{k-1}[k, j] \right) \]
  • \(D^k[i,j]\) : Shortest distance from \(i\) to \(j\) using vertices up to \(k\).
  • \(D^{k-1}[i,j]\) : The current best distance without using node \(k\).
  • \(D^{k-1}[i,k] + D^{k-1}[k,j]\) : Distance going from \(i\) to \(k\), and then \(k\) to \(j\).

Algorithm Pseudocode

To execute this across the whole graph, we use 3 nested loops:

for (k = 1 to V) { // Outer loop: the intermediate 'via' node for (i = 1 to V) { // Middle loop: the source row for (j = 1 to V) { // Inner loop: the destination column D[i][j] = min(D[i][j], D[i][k] + D[k][j]); } } }

9. Conclusion & Time Complexity

When should you use Floyd-Warshall?

  • Dense Graphs: It works exceptionally well when there are many edges connected to many nodes.
  • Negative Weights: Unlike Dijkstra, it correctly computes shortest paths when edges have negative costs (as long as there are no negative weight cycles).
Time Complexity: \(O(V^3)\)
Because of the three nested loops (k, i, j), the algorithm will always take \(V \times V \times V\) operations, where \(V\) is the number of vertices.
