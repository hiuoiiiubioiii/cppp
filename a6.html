1. The Setup: Memory Representation

We begin by declaring and initializing a 3x3 integer array. Assume an int takes 4 bytes, and the starting (base) address in memory is 100.

int a[3][3] = { {6, 2, 5}, {0, 1, 3}, {4, 9, 8} };

Logical vs. Physical View: Logically, we see a 3x3 grid. Physically, memory is a linear sequence of bytes. C stores 2D arrays in Row-Major Order (Row 0, then Row 1, then Row 2).

2. Core Concept: "An Array of Arrays"

A 2D array is technically a 1D array where each element is itself a 1D array.

  • a[0] is the name of the first 1D array (contains 6, 2, 5).
  • a[1] is the name of the second 1D array (contains 0, 1, 3).
  • a[2] is the name of the third 1D array (contains 4, 9, 8).
💡 Crucial C Rule: The name of an array acts as a pointer to its first element. Therefore, writing a[0] gives you the base address of the first row (100). Writing a[1] gives you the base address of the second row (112).

3. Pointer Initialization: Valid vs. Invalid

Let's declare an integer pointer: int *p;

❌ INVALID: p = a;

Why? a is an array of 1D arrays. Therefore, a decays into a pointer to an array of 3 integers int (*)[3]. However, p is just a pointer to a single integer int *. The types do not match.

✅ VALID: p = &a[0][0];

Why? a[0][0] is the actual integer value 6. Using & gets the address of that specific integer (100). Since it's the address of a single integer, p can hold it.

✅ VALID: p = a[0];

Why? a[0] is the name of the first 1D array. It decays to a pointer to its first element, which is the integer 6. It returns the address 100, which is compatible with int *p.

4. Different Ways to Print the Base Address

If you use printf("%u", ...); on the following expressions, they will ALL output 100, but structurally, they represent different pointer types:

  • p ➔ Holds the integer address 100.
  • a ➔ Pointer to the entire 1st row array. Value is 100.
  • &a[0][0] ➔ Address of the first specific integer. Value is 100.
  • &a ➔ Address of the entire 3x3 2D array. Value is 100.
  • a[0] ➔ Pointer to the first integer of the first row. Value is 100.
  • *a ➔ Dereferencing a gives the 1st row itself (a[0]). Value is 100.

5. Pointer Arithmetic on 2D Arrays

Because pointers represent different "sizes" of data, adding integers to them makes them jump by different amounts in memory.

What does a + 1 equal?

a points to a 1D array of 3 integers. Its size is 3 integers × 4 bytes = 12 bytes.
a + 1 means "skip one entire 1D array" (jump 12 bytes).
Result: 100 + 12 = 112 (Points to the start of the 2nd row).
Note: a + 1 is exactly equivalent to &a[1] and a[1].

What does *(a + 1) + 2 equal?

Let's break it down step-by-step:

  1. a + 1 points to the 2nd row (address 112).
  2. *(a + 1) dereferences it, giving us the array name a[1]. The array name acts as a pointer to its first integer. It is now an int* holding 112.
  3. + 2 means skip 2 integers (2 × 4 bytes = 8 bytes).
  4. Result: 112 + 8 bytes = 120 (Points to the 3rd integer in the 2nd row).

6. Accessing Values & The Golden Formula

How do we get the actual data value using pointers? We dereference the resulting address.

Example: *(*(a + 1) + 2)

  • From above, (*(a + 1) + 2) calculates the address 120.
  • Adding the outside * means "get the value at address 120".
  • Looking at our memory tape, the value at 120 is 3.
a[i][j]  =  *(*(a + i) + j)

The compiler translates standard bracket notation into this pointer arithmetic under the hood. Memorize this formula!

7. The Homework Questions

The instructor asked the following questions in the video:

Q1: **a = ?

Answer: 6.
a is the base address 100 (pointer to an array).
*a gives a[0], which evaluates to 100 (pointer to an integer).
**a dereferences the integer pointer at 100, yielding the value 6.

Q2: **p = ? (where p = a[0])

Answer: Compiler Error ❌.
p is an int*.
*p gives the integer value 6.
**p attempts to dereference the integer 6 (as if it were an address). You cannot dereference an integer, so this causes an error.

8. Deep Dive: a[1] + 1 vs &a[1] + 1

These look identical but jump completely different byte amounts because of the type of pointer they are.

a[1] + 1

a[1] points to the first integer of the 2nd row (address 112).

Adding 1 skips 1 integer (4 bytes).

Calculation: 112 + 4 = 116.

(Moves one column over)

&a[1] + 1

&a[1] points to the entire 2nd row (which is 12 bytes long).

Adding 1 skips the entire row (12 bytes).

Calculation: 112 + 12 = 124.
