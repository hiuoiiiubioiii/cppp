Graphs: Connected Components

A complete deep-dive into finding and counting connected components using Depth First Search (DFS).

1. Concept & Definitions

A Connected Component is defined as a subgraph of a main graph (the "super graph") that follows two strict conditions:

  • Reachable Vertices: Inside the component, any two vertices must be connected to each other by some path.
  • No Outside Connections: No vertex inside the component can have an edge connecting it to any vertex outside the component in the super graph.
Instructor's Tip: By simply looking at a drawing of a graph, a human can easily count how many isolated pieces exist. However, a computer cannot "see" it. We must write an algorithm to find them.

2. The Algorithm & Setup

To teach a computer to find components, we can use graph traversal algorithms like DFS (Depth First Search) or BFS (Breadth First Search). In this lecture, we use DFS.

Core Logic: The number of times the main recursive DFS function is triggered from the outer loop is exactly equal to the number of connected components in the graph.

The Target Graph (N = 9 Vertices)

We will trace our code against this specific 9-node graph drawn on the board. Visually, we can see there are 3 distinct components.

The Pseudo-Code

// Main Function to find components
Connected_Components(G) {
    // 1. Initialize all vertices as unvisited (-1)
    for each vertex v ∈ N {
        flag[v] = -1  
    }
    
    int count = 0; // Will hold the total components
    
    // 2. Loop through every vertex in the graph
    for(int v = 0; v < N; v++) {
        
        // 3. Only call DFS if the vertex is unvisited
        if(flag[v] == -1) {
            DFS(v, flag); // Call helper function
            count++;      // Increment component counter
            cout << "\n"; // Print newline for separation
        }
    }
}

// DFS Recursive Helper Function
DFS(int v, int flag[]) {
    flag[v] = 1;    // Mark current node as visited
    cout << v;      // Print the node
    
    // Check all neighbors of the current node
    for each adjacent node u of v {
        if(flag[u] == -1) {  // If neighbor is unvisited
            DFS(u, flag);    // Recursive call
        }
    }
}

3. Execution Trace: Component 1

Let's dry-run the algorithm on our 9-node graph.

N = 9
count = 0
flag = [-1,-1,-1,-1,-1,-1,-1,-1,-1]
Main Loop: v = 0

flag[0] == -1 is True. We enter the if-statement and call DFS(0).

  • DFS(0): Sets flag[0] = 1. Prints 0. Adjacents to 0 are {1, 3}. Takes adjacent u = 1.
  • flag[1] == -1. Calls DFS(1).
    • DFS(1): Sets flag[1] = 1. Prints 1. Adjacents are {0, 2}. 0 is visited. Takes u = 2.
    • flag[2] == -1. Calls DFS(2).
      • DFS(2): Sets flag[2] = 1. Prints 2. Adjacents are {1, 3}. 1 is visited. Takes u = 3.
      • flag[3] == -1. Calls DFS(3).
        • DFS(3): Sets flag[3] = 1. Prints 3. Adjacents are {0, 2}. Both are visited!
        • DFS(3) finishes and returns.
      • DFS(2) finishes and returns.
    • DFS(1) finishes and returns.
  • DFS(0) finishes and returns to main loop.

Back in main loop: count++ executes. Count is now 1. Newline is printed.

Console Output:
0 1 2 3

4. Execution Trace: Remaining Components

Skipping Visited Nodes

The main loop continues for v = 1, v = 2, and v = 3. Because they were fully explored above, their flag array values are all 1. The if-statement condition (flag[v] == -1) fails, so they are skipped.

count = 1
flag = [1, 1, 1, 1, -1, -1, -1, -1, -1]
Main Loop: v = 4

flag[4] == -1 is True. We call DFS(4).

  • DFS(4): Sets flag[4] = 1. Prints 4. Adjacent is {5}.
  • flag[5] == -1. Calls DFS(5).
    • DFS(5): Sets flag[5] = 1. Prints 5. Adjacent is {4} (visited).
    • DFS(5) finishes and returns.
  • DFS(4) finishes and returns to main loop.

Back in main loop: count++ executes. Count is now 2. Newline is printed.

Skipping Node 5

Main loop moves to v = 5. flag[5] is 1. Skipped.

Main Loop: v = 6

flag[6] == -1 is True. We call DFS(6).

  • DFS(6): Sets flag[6] = 1. Prints 6. Adjacents are {7, 8}. Takes u = 7.
  • flag[7] == -1. Calls DFS(7).
    • DFS(7): Sets flag[7] = 1. Prints 7. Adjacents are {6, 8}. Takes u = 8.
    • flag[8] == -1. Calls DFS(8).
      • DFS(8): Sets flag[8] = 1. Prints 8. Adjacents checked. All visited.
      • DFS(8) finishes and returns.
    • DFS(7) finishes and returns.
  • DFS(6) finishes and returns.

Back in main loop: count++ executes. Count is now 3. Newline is printed.

Loop moves to v = 7 and v = 8, both are skipped because they are visited. The main loop finishes (v < 9 is no longer true).
