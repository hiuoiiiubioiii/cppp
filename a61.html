AVL Tree Study Notes

Instructor Breakdown: Rules, Formulas, and Rotations

📌 Rule 1: It is a Binary Search Tree (BST)

Before you even think about balancing, an AVL tree is fundamentally a BST. You must follow these strict insertion rules:


  • Left Subtree: Every element here must be less than the parent node.
  • Right Subtree: Every element here must be greater than the parent node.
  • No Duplicates: As the instructor explicitly stated, duplicate elements are not allowed.

Alphabetical Data Example

The instructor uses variables x, y, z. Because of alphabetical order, we assume x < y < z. Therefore, 'y' goes to the right of 'x', and 'x' goes to the left of 'y'.

⚖️ Rule 2: The Balance Factor

This is what makes an AVL tree self-balancing. For every single node in the tree, you must calculate its Balance Factor.

The Formula

Balance Factor = Height(Left Subtree) - Height(Right Subtree)

  • Balanced set: {-1, 0, 1}. If a node results in one of these numbers, it is perfectly fine.
  • Unbalanced set: -2 or 2. The moment you see these, you MUST stop and rotate.

The Golden Rule of Insertion

Insert one element at a time. After a single insertion, recalculate the Balance Factor for all nodes. If an imbalance occurs, you must fix that specific subtree immediately before inserting any more data.

🔄 Case 1: Right-Right (RR) Insertion

Happens when you insert into the right subtree of the right child. The tree becomes a straight line slanting to the right.

Instructor's Math: Node 'x' has a left height of 0, and a right height of 2. BF = 0 - 2 = -2.

Solution: Perform a Left Rotation on 'x'. Pull 'x' down to the left.

x -2
y -1
z 0
Unbalanced RR
Left Rotation ➔
y 0
x 0
z 0
Balanced

🔄 Case 2: Left-Left (LL) Insertion

Happens when you insert into the left subtree of the left child. The tree becomes a straight line slanting to the left.

Instructor's Math: Node 'z' has a left height of 2, and a right height of 0. BF = 2 - 0 = 2.

Solution: Perform a Right Rotation on 'z'. Pull 'z' down to the right.

z 2
y 1
x 0
Unbalanced LL
Right Rotation ➔
y 0
x 0
z 0
Balanced

🛠️ Case 3: Right-Left (RL) Insertion

Happens when you insert into the left subtree of the right child. This creates a "kink" in the tree, making it impossible to fix with just one rotation.

Solution: A 2-step process. First, perform a Right Rotation on the bottom two nodes to straighten the line. Second, perform a Left Rotation on the root.

x -2
y 0
z 1
Unbalanced RL
1. Right Rot. (z) ➔
x -2
y -1
z 0
Straightened (RR)
2. Left Rot. (x) ➔
y 0
x 0
z 0
Balanced

🛠️ Case 4: Left-Right (LR) Insertion

Happens when you insert into the right subtree of the left child. Another "kink" scenario.

Solution: A 2-step process. First, perform a Left Rotation on the bottom two nodes to straighten it. Second, perform a Right Rotation on the root.

z 2
x -1
y 0
Unbalanced LR
1. Left Rot. (x) ➔
z 2
y 1
x 0
Straightened (LL)
2. Right Rot. (z) ➔
y 0

z 0
Balanced

🤫 The Instructor's Secret Exam Hack

Did you notice a pattern in all 4 cases above? If you ever get confused during an exam trying to rotate nodes in your head, remember this golden rule:

The Median Value ALWAYS Becomes the New Root.

Out of (x, y, z), 'y' is the middle value alphabetically. Look at every single final, balanced tree above. 'y' is always at the top, 'x' is on the left, and 'z' is on the right. Just find the median of the 3 nodes causing the imbalance, pop it to the top, and drop the other two down to the left and right!
