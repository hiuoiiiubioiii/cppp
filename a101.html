Merge Sort Algorithm

In-Depth Study Notes & Visualizations based on Jenny's Lectures

1. The Core Concept

Merge Sort is an algorithm that operates on the Divide and Conquer principle, similar to Quick Sort.

It consists of two massive phases:

  • Divide: The complete list is repeatedly divided into equal sub-lists until you get sub-lists containing only one element. (A list with one element is naturally sorted).
  • Merge: You repeatedly merge these single elements to produce new, sorted sub-lists until there is only one complete, sorted list remaining.

3. The MergeSort Function Code

How do we program the computer to do this? We write a recursive function.

MergeSort(A, lb, ub)
{
    if (lb < ub)  // Condition ensures at least 2 elements
    {
        mid = (lb + ub) / 2;
        
        // Recursive calls to Divide
        MergeSort(A, lb, mid);       // Sort left half
        MergeSort(A, mid + 1, ub);   // Sort right half
        
        // Combine the sorted halves
        Merge(A, lb, mid, ub);
    }
}

Note on Execution: The computer uses Depth-First execution. It doesn't divide everything left and right at the same time. It follows the function calls deeply down the left side until it hits a single element before it starts looking at the right side or merging.

Handling Exhaustion: Notice what happens at the end. When one pointer reaches the end of its list (e.g., j finishes), the other list still has numbers left. The algorithm simply copies all remaining elements directly into B.

6. The Merge Function Code

Here is the complete C-style logic for combining the arrays, representing exactly what you just visualized above.

Merge(A, lb, mid, ub)
{
    // Initialize pointers
    i = lb;
    j = mid + 1;
    k = lb;

    // Step 1: Compare and insert into temp array B
    while (i <= mid && j <= ub) 
    {
        if (A[i] <= A[j]) 
        {
            B[k] = A[i];
            i++; 
        } 
        else 
        {
            B[k] = A[j];
            j++;
        }
        k++;
    }

    // Step 2: Handle remaining elements if one array finishes early
    if (i > mid) // Left array is exhausted
    {
        while (j <= ub) 
        {
            B[k] = A[j];
            j++; k++;
        }
    } 
    else // Right array is exhausted
    {
        while (i <= mid) 
        {
            B[k] = A[i];
            i++; k++;
        }
    }

    // Step 3: Copy sorted elements back to original array A
    for (k = lb; k <= ub; k++) 
    {
        A[k] = B[k];
    }
}

7. Time & Space Complexity

To merge two arrays of size m and n, the time required is Θ(m + n).

Because the array is always divided completely in half regardless of its initial state, Merge Sort performs the exact same number of operations in all scenarios.

Complexity Type Value Reasoning
Best Case Time O(n log n) Array is always split and merged recursively.
Average Case Time O(n log n) Standard divide and conquer tree depth is log n, merging takes n.
Worst Case Time O(n log n) Even if reverse sorted, divisions and merges remain constant.
Space Complexity O(n) Requires the creation of the temporary auxiliary array B to hold elements while merging. (Not an in-place sort).
