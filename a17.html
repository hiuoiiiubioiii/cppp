Insertion in a Doubly Linked List

Complete Audio-Visual Breakdown & Study Notes (in C)

1. Core Concept: The tail Pointer

Before writing code, the instructor highlights a crucial design choice: maintaining a tail pointer alongside the head pointer.

What it is:

Just as head stores the memory address of the first node, tail stores the memory address of the last node.

The Pros vs. Cons:

  • Pro: Inserting or deleting at the very end becomes an O(1) time complexity operation. You don't have to traverse the list sequentially.
  • Con: Requires extra memory (4 bytes or 8 bytes) to store the pointer, and you must remember to update it when the last node changes.

2. Defining the Node Structure

A doubly linked list node is divided into three specific parts:

  1. prev: Pointer holding the address of the previous node.
  2. data: The actual value stored.
  3. next: Pointer holding the address of the next node.
// Global pointers initialized to 0 (NULL)
struct node *head = 0;
struct node *tail = 0;

struct node {
    int data;
    struct node *next;
    struct node *prev;
};

3. Creating the First Node (Empty List)

When the list is empty, both head and tail are NULL. When inserting the first node, both pointers must be updated to point to this new single node.

struct node *newnode;
// 1. Allocate memory dynamically
newnode = (struct node*)malloc(sizeof(struct node));

// 2. Ask user for data
printf("Enter data: ");
scanf("%d", &newnode->data);

// 3. Set isolated node pointers
newnode->prev = 0;
newnode->next = 0;

// 4. Update global pointers
if (head == 0) {
    head = tail = newnode;
}

4. Insertion at the Beginning (insertAtBeg)

Scenario: The list already exists. We want to push a new node to the very front.

Click "Step Forward" to trace the pointer updates.

The Logic:

  1. Create newnode and fill data. Set its prev = 0.
  2. Link forward: newnode->next = head.
  3. Link backward: head->prev = newnode.
  4. Shift head: head = newnode.
void insertAtBeg() {
    struct node *newnode;
    newnode = (struct node*)malloc(sizeof(struct node));
    scanf("%d", &newnode->data);
    
    newnode->prev = 0;
    newnode->next = head; 
    head->prev = newnode; 
    head = newnode; 
}

5. Insertion at the End (insertAtEnd)

Scenario: We want to add a node to the back. Thanks to the tail pointer, no traversal is needed!

Click "Step Forward" to see tail insertion.

The Logic:

  1. Create newnode and fill data. Set its next = 0.
  2. Link backward: newnode->prev = tail.
  3. Link forward: tail->next = newnode.
  4. Shift tail: tail = newnode.
void insertAtEnd() {
    struct node *newnode;
    newnode = (struct node*)malloc(sizeof(struct node));
    scanf("%d", &newnode->data);
    
    newnode->next = 0;
    newnode->prev = tail; 
    tail->next = newnode;
    tail = newnode; 
}

6. Insertion at a Specific Position (insertAtPos)

Scenario: User inputs pos = 3. We want to insert after the 2nd node. We must traverse to pos - 1.

Traversing to position - 1. Click "Step Forward".

The 4 Critical Pointer Updates (Order matters!):

Assuming temp has stopped at the node just before insertion:

  1. newnode->prev = temp; (New node looks back at temp)
  2. newnode->next = temp->next; (New node looks forward)
  3. temp->next->prev = newnode; (Next node looks back to new node)
  4. temp->next = newnode; (Temp looks forward to new node)
void insertAtPos() {
    int pos, i = 1;
    scanf("%d", &pos);
    
    // (Assume pos validity checks are done here)
    
    struct node *newnode, *temp;
    temp = head; 
    
    newnode = (struct node*)malloc(sizeof(struct node));
    scanf("%d", &newnode->data);
    
    // Traverse to node at pos - 1
    while (i < pos - 1) {
        temp = temp->next;
        i++;
    }
    
    // The 4 critical pointer links
    newnode->prev = temp;              
    newnode->next = temp->next;        
    temp->next->prev = newnode;        
    temp->next = newnode;              
}

7. Time Complexity Cheat Sheet

Operation Time Complexity Reason
Insert at Beginning O(1) Directly manipulates the head pointer. No traversal needed.
Insert at End (WITH tail) O(1) Directly manipulates the tail pointer. No traversal needed.
Insert at End (WITHOUT tail) O(n) Must use a while loop to traverse the entire list to find the last node.
Insert at Position O(n) Must traverse the list sequentially to find the specific position.
