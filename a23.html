📚 Master Class: Deletion from a Circular Linked List

Complete student breakdown covering the logic, visual pointers, and C code for deleting nodes from the Beginning, End, and a Specific Position.

🔑 The Golden Rule & Setup

Before we start deleting, remember how the instructor setup the board:

  • We are NOT using a head pointer. We are only maintaining a tail pointer.
  • How do we find the first node? The next part of the tail node always contains the address of the first node.
  • Formula: First Node = tail->next;

Standard Node Structure

struct node {
    int data;
    struct node *next;
} *tail; // Global tail pointer

🗑️ Part 1: Deletion from the Beginning

The Goal: Remove the very first node in the list and free its memory.

The Logic

  1. Create a temporary pointer (temp) pointing to the first node (tail->next).
  2. Update the tail to point to the second node (tail->next = temp->next).
  3. Free the memory of the first node (temp).

C Code

void delFromBeg() {
    struct node *temp;
    temp = tail->next; // temp points to the 1st node

    if (tail == 0) { 
        printf("List is empty");
    } 
    else if (temp->next == temp) { // Only ONE node exists
        tail = 0;
        free(temp);
    } 
    else { 
        tail->next = temp->next; // Tail bypasses the 1st node
        free(temp);              // Clean up garbage memory
    }
}

🗑️ Part 2: Deletion from the End

The Goal: Remove the last node (the tail node) and update the tail pointer to the second-to-last node.

The Logic

  1. Because it's a singly linked list, we can't go backward. We must traverse from the beginning.
  2. Create current (starts at head) and previous (trails one step behind).
  3. Traverse until current->next == tail->next (meaning current hit the last node).
  4. Make previous->next = tail->next to complete the circle, update tail = previous, and free(current).

C Code

void delFromEnd() {
    struct node *current, *previous;
    current = tail->next; 

    if (tail == 0) {
        printf("List is empty");
    } 
    else if (current->next == current) { 
        tail = 0;
        free(current);
    } 
    else {
        // Traverse to the end
        while (current->next != tail->next) { 
            previous = current;       
            current = current->next;  
        }
        
        previous->next = tail->next; // Link 2nd-to-last to First
        tail = previous;             // Update tail pointer
        free(current);               // Free the old tail
    }
}

🗑️ Part 3: Deletion from a Specific Position

The Goal: User enters a position (e.g., 3). Traverse to position 2, bypass position 3, and free the memory.

The Logic

  1. Get the total length of the list (using a helper function getLength()) to validate user input.
  2. Traverse the list using a while loop. Stop exactly ONE node before the target position (i < pos - 1).
  3. Create nextnode = current->next to hold the node being deleted.
  4. Bypass the node: current->next = nextnode->next.
  5. Free nextnode.

C Code

void delFromPos() {
    struct node *current, *nextnode;
    int pos, i = 1, l;

    current = tail->next; 
    
    printf("Enter position: ");
    scanf("%d", &pos);
    
    l = getLength(); // Helper function

    if (pos < 1 || pos > l) { 
        printf("Invalid Position");
    } 
    else if (pos == 1) { 
        delFromBeg(); 
    } 
    else {
        // Stop ONE node BEFORE the target
        while (i < pos - 1) {
            current = current->next;
            i++;
        }
        
        nextnode = current->next; // Node to be deleted
        
        current->next = nextnode->next; // Bypass the node
        free(nextnode); // Free memory
    }
}

📝 Final Study Tips

Two major takeaways from the instructor's breakdown:

  • Draw the Boxes: When tracking circular lists on an exam, always draw the data/address boxes. Write the actual address numbers (like 100, 500) inside the "next" sections. It makes pointer logic foolproof.
  • Order of Operations Matters: Always update the links of the surviving nodes before you call free(). If you free() the target node first, you lose its ->next data, breaking the chain!
