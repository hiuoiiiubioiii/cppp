1. What is a Heap?

A Heap is a special tree-based data structure that satisfies two main conditions:

  • Shape Property: It must be an Almost Complete Binary Tree. This means all levels of the tree are completely filled, except possibly the last level. The last level must be filled strictly from left to right.
  • Heap Property: The value of parent nodes must follow a specific ordering relative to their children (either Max or Min).

2. Array Representation of a Binary Tree

Even though we draw heaps as trees, they are almost always stored in memory as Arrays. This saves space (no pointers needed) and makes finding parents/children extremely fast using simple math.

The Math Formulas:
If a node is placed at index i (assuming the root is at i = 1):
  • Left Child: is at index 2 * i
  • Right Child: is at index (2 * i) + 1
  • Parent: is at index floor(i / 2)

3. Max Heap vs. Min Heap

Depending on how we order the parent vs. child nodes, a heap can be a Max Heap or a Min Heap.

Max Heap

Every parent node must be greater than or equal to its child nodes.

A[Parent[i]] >= A[i]

Rule: The Root node will ALWAYS be the largest element.

4. Insertion in a Max Heap

Time Complexity: O(log n)

Algorithm:

  1. Increase the array size n = n + 1.
  2. Insert the new value at the very end of the tree (to maintain the Complete Binary Tree shape).
  3. Heapify Up: Compare the newly inserted node with its parent. If the child is greater than the parent, swap them. Repeat until the property is restored or you reach the root.
Pseudocode: 
insertHeap(A, n, value) {
  n = n + 1;
  A[n] = value;
  i = n;
  
  while (i > 1) {
    parent = floor(i / 2);
    
    if (A[parent] < A[i]) { 
      swap(A[parent], A[i]);
      i = parent;
    } else {
      return; 
    }
  }
}

5. Deletion in a Max Heap

Crucial Rule: You can ONLY delete the Root Node (the maximum element). You cannot delete a leaf or an arbitrary internal node directly.

Time Complexity: O(log n)

Algorithm:

  1. Remove the Root element.
  2. Take the last element in the tree and move it to the root position.
  3. Reduce the array size n = n - 1.
  4. Heapify Down: Compare the new root with its left and right children. Find the largest of the two children. If the parent is smaller than the largest child, swap them. Repeat down the tree.

6. Bonus: Introduction to Heap Sort

Think about what happens when you perform deletion repeatedly:

  • Every time you delete, you extract the largest element.
  • The size of your heap shrinks by 1, leaving an empty spot at the very end of your array.
  • If you save the deleted element into that newly freed spot at the end of the array...
  • Magic! After deleting all elements, your array will be perfectly sorted in Ascending Order.

This clever technique is the foundation of the Heap Sort algorithm.
