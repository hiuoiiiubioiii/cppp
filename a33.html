1. Core Concepts

Understanding Expressions

Infix Expression

The operator is written between the operands. This is how humans read math.

A + B

Postfix Expression

The operator is written after the operands. This is easier for computers to process.

A B +

Why use a Stack?

Converting manually using BODMAS requires scanning the expression back and forth multiple times. The Stack Algorithm allows a computer to convert the expression efficiently in a single scan from left to right.

3. The Rules

How the Algorithm Works

Scan the expression from left to right. Apply these rules based on the symbol you read:

1

If it's an Operand (Letter/Number)

Print it directly to the Postfix output. Operands never go into the stack.

2

If it's an Operator (+, -, *, /, ^)

Compare it with the operator currently at the Top of the Stack.

Higher Precedence: Push incoming operator to the stack.
Lower Precedence: Pop the stack top to the output. Repeat comparison until incoming > top, then Push.
Same Precedence: Check Associativity. If Left-to-Right -> Pop then Push. If Right-to-Left -> Push.
3

Parentheses Handling

If Opening `(` : Push it to the stack.
If Closing `)` : Pop operators from stack to output until you hit the `(`. Discard both parentheses.
