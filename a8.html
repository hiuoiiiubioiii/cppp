Types of Linked Lists

Comprehensive Video Lecture Breakdown & Visualizations

Introduction

In this lecture, the instructor introduces the four standard types of linked lists. However, she notes a crucial concept: you can modify these base structures according to your specific programming needs. They are not strictly rigid.

The four fundamental types we will cover are:

  • Singly Linked List
  • Doubly Linked List
  • Circular Linked List
  • Doubly Circular Linked List

1. Singly Linked List

Concept: This is the most common type. If someone just says "Linked List," they usually mean a Singly Linked List. It is called "singly" because there is only a single link between nodes.

Because of this single link, only forward, sequential traversal is possible. You cannot move backward through the list.

Instructor's Crucial Note: The head is not a node. It is simply a pointer variable whose only job is to store the starting address of the first node so the program knows where the list begins.

Code Representation (C-Style Struct)

struct node {
    int data;               // Stores the actual value
    struct node *next;      // Pointer to the next node
};

2. Doubly Linked List

Concept: As the name suggests, this list has double links (two pointers per node). Because it has two pointers, you can traverse the list in both forward and backward directions.

Each node consists of three parts: a pointer to the previous node, the actual data, and a pointer to the next node.

Code Representation

struct node {
    int data;               // Stores the actual value
    struct node *next;      // Pointer to the succeeding node
    struct node *prev;      // Pointer to the preceding node
};

3. Circular Linked List

Concept: A Circular Linked List is a direct variation of the Singly Linked List. The only difference is that the list forms a continuous circle. There is no Null at the end of the list; the last element links back to the first element.

Notice that the structure code is exactly the same as the Singly Linked List! A circular list is created purely through logic during implementation (by making the last node point to the head node rather than Null).

4. Doubly Circular Linked List

Concept: This is a combination of both a Doubly Linked List and a Circular Linked List. It has two pointers per node (forward and backward). There are no Null values at either the beginning or the end of the list. The list is completely connected in a circle in both directions.

Just like before, the struct definition is exactly the same as the standard Doubly Linked List. It becomes Circular during implementation when you assign the first node's prev to the last node, and the last node's next to the first node.

Summary & Next Steps

The instructor concludes by noting that while Doubly Linked Lists allow backward traversal, they come at the cost of requiring more memory space to store the extra pointer in every single node.
