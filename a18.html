Doubly Linked List: Deletion

A complete visual breakdown of the instructor's lecture. Scroll down to review the code, logic, and visual pointer movements.

Instructor's Golden Rule: The massive advantage of a Doubly Linked List over a Singly Linked List is that you only need one pointer (temp) to delete a node from the middle! You don't need to trail behind with a second pointer because every node already knows its prev neighbor.

1. Deletion from the Beginning

Goal: Remove the very first node and free its memory without breaking the rest of the list.

  • We cannot just do head = head->next immediately. If we do, we lose the memory address of the first node forever (creating a memory leak).
  • We must put a temp pointer on the first node.
  • Move head to the second node.
  • Set the new head's prev pointer to 0 (NULL).
  • Free temp.

void delFromBeg() {
    struct node *temp;

    // 1. Check for Underflow
    if (head == 0) {
        printf("List is empty");
    } else {
        // 2. Point temp to the node to be deleted
        temp = head; 

        // 3. Shift head forward
        head = head->next; 

        // 4. Break the backward link of the new first node
        head->prev = 0; 
        
        // 5. Clean up garbage memory
        free(temp); 
    }
}

⏱️ Time Complexity: O(1). We directly access the node via the `head` pointer; no traversal needed.

2. Deletion from the End

Goal: Remove the very last node. Because we maintain a tail pointer, this is incredibly fast.

  • Point temp to tail.
  • Move tail backward using tail = tail->prev.
  • Make the new tail's next pointer 0 (NULL) to detach the last node.
  • Free temp.

void delFromEnd() {
    struct node *temp;

    if (tail == 0) {
        printf("List is empty");
    } else {
        // 1. Temp tracks the node we want to kill
        temp = tail;

        // 2. Tail steps backwards
        tail = tail->prev;

        // 3. Break the forward link to the dying node
        tail->next = 0;
        
        // 4. Free memory
        free(temp);
    }
}

⏱️ Time Complexity: O(1). Because of the `tail` pointer, we instantly access the end of the list without loops.

3. Deletion from a Specific Position

Goal: Delete a node at a specific index given by the user (e.g., position 3). This is where the power of the Doubly Linked List shines.

If we want to delete Node 3, we need to connect Node 2 directly to Node 4. Here is the instructor's logic decoded:

  • Traverse with temp until it sits on position 3.
  • temp->prev is Node 2. We change its `next` to point to Node 4: temp->prev->next = temp->next;
  • temp->next is Node 4. We change its `prev` to point to Node 2: temp->next->prev = temp->prev;
  • Free temp.

    struct node *temp;

    printf("Enter Position: ");
    scanf("%d", &pos);

    // Note: Add if(pos==1) delFromBeg() logic here in reality

    temp = head;

    // 1. Traverse to the node
    while (i < pos) {
        temp = temp->next;
        i++;
    }

    // 2. The Magic Bypass Steps:
    // Look backwards, update its forward pointer
    temp->prev->next = temp->next;

    // Look forward, update its backward pointer
    temp->next->prev = temp->prev;

    // 3. Free the bypassed node
    free(temp);
}

⏱️ Time Complexity: O(n). Because we have to loop through the list to find the specific position, execution time grows linearly with the position size.
