1. The Base Graph & Adjacencies

Traversing a graph means visiting every vertex (node) in the graph systematically. The instructor provides an example graph with 7 nodes (0 through 6). Before we traverse, we must understand how they are connected.

Adjacency List (Who is connected to who?)

Node 0 -> 1, 3
Node 1 -> 0, 3, 2, 5, 6
Node 2 -> 1, 3, 4, 5, 6
Node 3 -> 0, 1, 2, 4
Node 4 -> 3, 2, 6
Node 5 -> 1, 2
Node 6 -> 1, 2, 4

Note: The instructor states you can start a traversal from ANY node. For these examples, we will use Node 0 as our starting Root Node.

Breadth-First Search (BFS) Concept

Also known as Level-Order Traversal. We explore all adjacent nodes (the closest neighbors) of the current node before moving deeper.

  • Data Structure Used: Queue (FIFO - First In, First Out)
  • Golden Rule: Only insert adjacent nodes into the queue if they are unvisited and not already in the queue.

Insert 0 into Queue. Pop 0 to Result. Mark 0 visited. Find adjacencies of 0: (1, 3). Insert them into Queue.

Pop 6. (Adjacencies 1, 2, 4 are visited/in queue). Insert nothing.

Depth-First Search (DFS) Concept

We go as deep as possible down one path. When we hit a "dead end" (no unvisited neighbors), we backtrack to the previous node and try a different path.

  • Data Structure Used: Stack (LIFO - Last In, First Out)
  • Golden Rule: Pick only ONE unvisited adjacent node at a time. Push it to the stack, mark it visited, and immediately dive into its neighbors.

DFS Step-by-Step Walkthrough

Step 1 to 4: Diving Deep

Start 0. Push 0. Pick adj 1. Push 1.
From 1, pick adj 3. Push 3.
From 3, pick adj 2. Push 2.
From 2, pick adj 5. Push 5.

Step 5: The First Dead End!

Look at Node 5. Its adjacencies are 1 and 2. Both are already visited! We cannot go deeper. We must backtrack. Pop 5 off the Stack.

Step 6 & 7: Backtrack and Dive Again

Backtrack to 2. Look at 2's adjacencies. 4 and 6 are unvisited. Pick 4. Push 4.
From 4, look at adjacencies. 6 is unvisited. Pick 6. Push 6.

Step 8: Final Backtracking

Look at Node 6. Adjacencies 1, 2, 4 are all visited. Dead end. Pop 6.
Backtrack down the stack: 4 (all visited) -> Pop 4.
2 (all visited) -> Pop 2. ... Pop 3. Pop 1. Pop 0. Stack empty!

Stack State: Empty
Final DFS: 0, 1, 3, 2, 5, 4, 6

⚠️ Instructor's Golden Note

"This is not the ONLY valid answer!"

Because nodes have multiple neighbors, the order you choose to visit them changes the final result. For example, in DFS Step 1, if we chose to explore Node 3 instead of Node 1, the entire sequence would change. Both answers would be 100% correct as long as you follow the structural rules of Queues (BFS) and Stacks (DFS).
