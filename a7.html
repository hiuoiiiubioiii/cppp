Concept 1

The Memory Manager

Computer memory is like a long tape of bytes. Each byte has a unique, sequential address (like 100, 101, 102...).

It is a crucial and limited resource managed by the Operating System's Memory Manager, which decides which processes get memory and where.

Problem

The Drawback of Arrays

Arrays require contiguous (continuous) memory and have a fixed size. This leads to two major issues:

  • Scenario A (Expensive Resizing): If you create a[3] and later need a 4th slot, you can't just attach it. The space next to it might be occupied. The manager has to find a whole new 16-byte block elsewhere, copy the data, and delete the old block.
  • Scenario B (Wasted Space): To avoid resizing, you might declare a huge array just in case: int a[50];. If you only store 10 items, 40 slots (160 bytes) are entirely wasted and blocked from other programs.

Solution

Enter the Linked List (Nodes)

A Linked List solves this by storing data wherever free memory is available. Items do not need to be next to each other!

The Challenge: If items are scattered, how do we find the next one?
The Fix: Each element must store extra information—a pointer with the address of the next element. This combo (Data + Pointer) is called a Node.

Code

Declaring a Node in C

You cannot use standard data types (like int) to create a node because a node holds two different types of data: an integer and an address/pointer.

Therefore, we use a user-defined data type: struct. The pointer inside doesn't point to a standard integer; it points to the entirety of the next node.

struct node {
    int data;             // The actual data (e.g., 4)
    struct node *next;    // Pointer to the next node
};
Concept 2

Traversal & The "Head" Pointer

Because nodes are scattered randomly in memory, you can't just access the 3rd item instantly like you can in an array (a[2]).

You must have a starting point. This is the head pointer. It is a dedicated variable that only stores the address of the very first node.

The Trade-off: To find the 10th element, you must start at the head and follow 10 links sequentially. This makes searching slow. Time complexity for accessing data is O(n), compared to O(1) in arrays.

Advantage

The Superpower: Insertion & Deletion

While arrays are fast at searching, they are terrible at insertion and deletion. Linked lists excel here.

Array Insertion: O(n)

Inserting at the beginning requires shifting every single element to the right to make room.
