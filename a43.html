Queue Implementation using Linked List

Complete Video Breakdown & Study Feed

1. The Problem with Arrays

In the previous lesson, we implemented a Queue using an Array. However, arrays have a major limitation:

  • Static Memory Allocation: You must fix the maximum size of the queue at the beginning (e.g., size = 5).
  • Overflow & Copying: If you try to insert a 6th element, it overflows. To fix this, you have to create a new, larger array and copy all old elements over.
  • Time Complexity Flaw: Copying elements takes O(N) time.
The Solution: Dynamic Memory Allocation
By using a Linked List, memory is allocated on the fly using malloc(). The queue can grow and shrink infinitely (until RAM runs out) without ever needing to copy old elements.

2. The "Aha!" Concept: Why We Need Two Pointers

A Queue strictly follows the FIFO (First In, First Out) principle. Furthermore, both Enqueue (Insertion) and Dequeue (Deletion) MUST happen in O(1) constant time.

Attempt 1: Standard Linked List (One Pointer)

If we use a standard linked list with only a head pointer:

  • Deletion (Front): head = head->next. This takes O(1). ✅
  • Insertion (Rear): You must traverse the entire list from the head to find the last node. This takes O(N). ❌ This breaks the rules of a Queue!

Attempt 2: The Solution (Two Pointers)

To fix this, we maintain TWO pointers. We replace head with front and rear.

3. Code Setup: Structures & Global Variables

First, we define what a single node looks like in C, and declare our two global pointers initialized to 0 (NULL), representing an empty queue.

struct node {
    int data;
    struct node *next;
};

// Global pointers initialized to NULL
struct node *front = 0;
struct node *rear = 0;

4. Enqueue Operation (Insertion)

Insertion always happens at the rear. We use malloc to dynamically create a new memory block.

void enqueue(int x) {
    // 1. Create a new node
    struct node *newnode;
    newnode = (struct node *)malloc(sizeof(struct node));
    newnode->data = x;
    newnode->next = 0;

    // 2. Condition A: If the queue is empty
    if (front == 0 && rear == 0) {
        front = rear = newnode;
    } 
    // 3. Condition B: Queue has elements
    else {
        rear->next = newnode; // Link old rear to new node
        rear = newnode;       // Move rear pointer to new node
    }
}

5. Displaying the Queue

To print the queue without modifying front or rear, we use a temporary pointer temp.

void display() {
    struct node *temp;
    
    if (front == 0 && rear == 0) {
        printf("Queue is empty");
    } else {
        temp = front;
        while (temp != 0) {
            printf("%d ", temp->data);
            temp = temp->next; // Move temp forward
        }
    }
}

6. Dequeue Operation (Deletion)

Deletion always happens at the front. We must carefully update the front pointer and explicitly free the memory to prevent memory leaks.

void dequeue() {
    struct node *temp;
    temp = front; // Store the node to be deleted

    if (front == 0 && rear == 0) {
        printf("Queue is empty");
    } else {
        printf("Dequeued element: %d", front->data);
        front = front->next; // Move front to the next node
        free(temp);          // IMPORTANT: Free memory!
    }
}
Instructor Note on `free()`: In dynamic memory allocation, leaving unlinked nodes floating in memory creates garbage. You must explicitly use the free(temp) function to clear up the RAM.

7. Peek Operation

Peek simply looks at the front element without deleting it.

void peek() {
    if (front == 0 && rear == 0) {
        printf("Queue is empty");
    } else {
        printf("%d", front->data);
    }
}

✅ Lecture Summary Checklist

  • Dynamic Allocation: Overcomes the static size limitations of arrays.
  • Why Two Pointers? A single head pointer results in O(N) insertion. Adding a rear pointer makes it O(1).
  • Enqueue: Handled by updating rear->next and moving rear.
  • Dequeue: Handled by moving front to front->next and using free(temp).
  • Time Complexity: Enqueue, Dequeue, and Peek are all achieved in exactly O(1) time.
