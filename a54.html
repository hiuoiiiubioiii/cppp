3. Writing the Preorder Code

We are using a recursive approach. A recursive function must have a base condition to terminate, otherwise it will result in an infinite loop.

// Main function starting the traversal
void main() {
    struct node *root;
    // ... tree creation logic ...
    printf("Preorder is : ");
    preorder(root); // Passes address 100
}

// The Recursive Function (Numbered for the Dry Run)
void preorder(struct node *root) {
1.  if (root == 0)            // Base/Termination condition
2.      return; 
3.  printf("%d, ", root->data); // Process ROOT
4.  preorder(root->left);       // Process LEFT
5.  preorder(root->right);      // Process RIGHT
}

Why line 1 & 2? When we reach a leaf node (like 7 at address 400), it has no left child. Its `left` pointer is `0`. The function calls `preorder(0)`. If we didn't have line 1, line 3 would try to print `0->data`, which causes a segmentation fault (crash).

4. The Ultimate Dry Run (Stack Trace)

This is the most critical part of the lecture. Every time a function is called, memory is allocated in the Stack. When a function hits return, that stack frame is destroyed, and we resume the previous function exactly where we left off.

  • 3. printf(400->data) -> OUTPUT: 7

1. if(0 == 0) -> True!
2. return; (Frame destroyed instantly)

Back to Frame 2: preorder(300)

Resume

5. Coding Inorder & Postorder

Because we understand the recursive stack, changing the traversal order is incredibly simple. All we do is move the printf statement to match the formula.

Inorder (Left → Root → Right)

void inorder(struct node *root) {
  if (root == 0) return; 

  inorder(root->left);

  // Root is printed in the middle
  printf("%d, ", root->data);
    
  inorder(root->right);
}

Postorder (Left → Right → Root)

void postorder(struct node *root) {
  if (root == 0) return; 

  postorder(root->left);
  postorder(root->right);

  // Root is printed at the end
  printf("%d, ", root->data);
}
