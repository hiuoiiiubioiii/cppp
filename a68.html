Insertion in Splay Trees

Complete breakdown combining instructor's verbal theory, whiteboard visual walkthroughs, and algorithmic pseudocode.

📌 1. The Core Concept: The Two-Step Rule

The most important rule to remember for Splay Trees is that all basic operations rely on the standard Binary Search Tree (BST) logic, followed by one crucial extra step.

Step 1: Perform the standard BST operation (in this case, normal BST insertion at a leaf node).
Step 2: Perform the Splaying Operation on the newly inserted node.

What is Splaying?

Splaying is the process of bringing a specific node to the Root of the tree using a series of specific rotations (Left and Right). We do this so that recently accessed or inserted elements are closer to the root, making future accesses much faster.

💻 3. Algorithm Breakdown: `Insert(T, n)`

Here is the exact pseudocode written on the whiteboard. Let T be the tree, and n be the new node.

// T = Tree, n = New Node to be inserted
function Insert(T, n) {
    var temp = T.root;       // Pointer to traverse
    var y = NULL;            // Trailing pointer (will be parent)

    // Step 1: Standard BST Traversal
    while (temp != NULL) {
        y = temp;
        if (n->data < temp->data) {
            temp = temp->left;
        } else {
            temp = temp->right;
        }
    }

    // Step 2: Attach new node
    n->parent = y;

    // Step 3: Handle empty tree vs left/right child
    if (y == NULL) {
        T.root = n;
    } else if (n->data < y->data) {
        y->left = n;
    } else {
        y->right = n;
    }

    // Step 4: The magic SPLAY operation!
    Splay(T, n);
}

💻 4. Algorithm Breakdown: `Splay(T, n)`

This is the core logic that loops until the inserted node `n` becomes the root.

function Splay(T, n) {
    while (n->parent != NULL) {
        
        // ZIG / ZAG: Parent is Root (No Grandparent)
        if (n->parent == T.root) {
            if (n == n->parent->left) {
                right_rotation(T, n->parent);
            } else {
                left_rotation(T, n->parent);
            }
        } 
        else {
            var p = n->parent;
            var g = p->parent;

            // ZIG-ZIG (Left-Left)
            if (n == p->left && p == g->left) {
                right_rotation(T, g);  // Grandparent FIRST
                right_rotation(T, p);  // Parent SECOND
            }
            // ZAG-ZAG (Right-Right)
            else if (n == p->right && p == g->right) {
                left_rotation(T, g);   
                left_rotation(T, p);   
            }
            // ZIG-ZAG (Right-Left Triangle)
            else if (n == p->right && p == g->left) {
                left_rotation(T, p);   // Parent FIRST
                right_rotation(T, g);  // Grandparent SECOND
            }
            // ZAG-ZIG (Left-Right Triangle)
            else { 
                right_rotation(T, p);  
                left_rotation(T, g);   
            }
        }
    }
}

🔁 5. Helper Functions: Rotations

The fundamental left and right rotation logic as written on the board. Note: In full C/C++ implementations, you must also carefully update `parent` pointers.

// Right Rotation on node x
function right_rotation(T, x) {
    var y = x->left;        // y is left child
    x->left = y->right;     // y's right subtree becomes x's left
    y->right = x;           // x drops down to become y's right child
    return y;               // y is the new local root
}

// Left Rotation on node x
function left_rotation(T, x) {
    var y = x->right;       // y is right child
    x->right = y->left;     // y's left subtree becomes x's right
    y->left = x;            // x drops down to become y's left child
    return y;               // y is the new local root
}
