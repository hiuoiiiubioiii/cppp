1. Introduction to Splay Trees

Splay trees are a fascinating variant of Binary Search Trees (BST). Before diving into Splay Trees, you must understand standard BSTs (where the left child is smaller, right child is greater).

How are they different? Unlike AVL trees or Red-Black (RB) trees which are strictly balanced, Splay trees are roughly balanced (or self-adjusting). They don't enforce strict height rules after every insertion.

2. Time Complexity & Motivation

Why do we need Splay Trees if AVL and Red-Black trees already exist?

O(n) <-- Worst case for standard BST (skewed)

O(log n) <-- Guaranteed by AVL / RB Trees
  • Standard BST: In the worst case (e.g., inserting data in already sorted order), the tree becomes a straight line (skewed). Finding an element takes O(n).
  • Strictly Balanced (AVL/RB): By strictly maintaining height, these guarantee O(log n) for search, insert, and delete.
The Big Question: Can we achieve better than O(log n) in practical, real-world situations? The answer is Yes, using Splay Trees!

3. The Core Concept: Splaying

The standard operations of a Splay Tree are exactly the same as a normal BST. However, they possess one extra, defining step.

Operations: search, insert, delete

Search + splaying
Insert + splaying
Delete + splaying

What is Splaying? Splaying is the process of rearranging the tree so that the element you just operated on (searched, inserted, or deleted) becomes the new root of the entire tree. We achieve this through specific rotations.

4. The Rotation Rules (How to Splay)

To move our target node to the root, we analyze its relationship with its Parent and Grandparent. There are three primary cases.

Case 1: Child of the Root Node (Zig / Zag)

If the node you are searching for is a direct child of the root (it has a parent, but no grandparent), you perform a single rotation.

  • If it's a left child: Perform a Right Rotation (Zig)
  • If it's a right child: Perform a Left Rotation (Zag)

Case 2: "Same Side" (Zig-Zig / Zag-Zag)

The target node has a Parent and Grandparent. Both the Target and Parent are Left children, OR both are Right children (it forms a straight line).

Crucial Rule: First rotate the Parent over the Grandparent. Second, rotate the Target over the Parent.

Case 3: "Opposite Sides" (Zig-Zag / Zag-Zig)

The target node has a Parent and Grandparent. The Parent is a left child, but the Target is a right child (or vice versa). It forms a dog-leg or zigzag shape.

Crucial Rule: First rotate the Target over the Parent. Second, rotate the Target over the Grandparent.

5. Advantages, Drawbacks & Applications

- no extra info stored
- fastest type of BST for many practical applications
- easy implementation
- better performance because frequently accessed nodes will move nearer to root
- O(n) worst case, O(log n) amortized
- caches

Advantages:

  • Space Efficient: You don't need to store extra data in the nodes (like height integers in AVL, or color bits in Red-Black trees).
  • Easy Implementation: It relies entirely on standard BST algorithms plus basic tree rotations.
  • Locality of Reference: Data accessed recently is highly likely to be accessed again. Because splaying puts accessed data at the root, the next search for that same data takes O(1) time!

Drawbacks:

  • Because it is not strictly balanced, if data is inserted sequentially (1, 2, 3, 4...), the tree becomes a straight line, resulting in an O(n) worst-case time for a single operation. However, its amortized (average) time over multiple operations remains O(log n).
Real-World Application: CACHES! Because frequently accessed items naturally bubble to the top of the tree (becoming the root), Splay Trees are perfect for implementing Cache Memory, network routers, and memory allocators.

Instructor's Challenge / Homework

At the end of the video, she points to a node structure: 10 (root) → Right to 15 → Left to 13 → Right to 14.

"Find 14. Which operations/rotations do you need to perform to make 14 the root?"

Solution Breakdown:

  • Target = 14. Parent = 13. Grandparent = 15.
  • 13 is a Left child. 14 is a Right child. They are on opposite sides.
  • This is Case 3 (Zig-Zag).
  • Step 1: Left Rotate (Zag) 14 over 13.
  • Step 2: Right Rotate (Zig) 14 over 15.
  • Once 14 moves up, it will have 10 as its new parent. You repeat the process until 14 hits the root!
