Representation of a Graph

A continuous, in-depth breakdown of how computers store pictorial graphs in memory.

1 The Problem & The Example Graph

While humans can easily understand a visual drawing of a graph, computers require specific mathematical data structures to process and store this information. The two most popular industry-standard methods are the Adjacency Matrix and the Adjacency List.

Throughout this lecture, we will use the following undirected graph as our reference point. It has 5 vertices (nodes).

2 Method 1: Adjacency Matrix

An Adjacency Matrix uses a 2D array to represent edges. The dimensions are always an n × n matrix, where n is the total number of vertices.

Formal Mathematical Definition:
For a matrix A[n][n]:
A[i][j] = 1 IF node i and node j are adjacent (an edge exists).
A[i][j] = 0 OTHERWISE (no edge exists).
  • The Diagonal (Loops): We first check if a node connects to itself. Because our graph has no loops, the entire main diagonal (1-1, 2-2, 3-3...) is 0.
  • Symmetry: Because this is an undirected graph, if 1 connects to 2, 2 connects to 1. The matrix will be a mirror image across the diagonal.

3 Method 2: Adjacency List

An Adjacency List dynamically allocates memory using an Array of Linked Lists.

First, you create a 1-Dimensional array equal to the number of vertices (n = 5). Each index acts as a "head pointer" for a Linked List. The linked list attached to array index X contains a sequence of all nodes directly connected to node X.

4 Space Complexity Analysis

How much physical computer memory does each method require to store the graph?

Adjacency Matrix Space

Θ(n²)

Because it requires an n × n grid regardless of how many edges actually exist (in our case, 25 memory blocks).

Adjacency List Space

Θ(n + 2e)

n is the initial array size. 2e (double the edges) is because it's an undirected graph—every edge is recorded twice (e.g., 1→2 and 2→1).

5 Matrix vs. List: Which to choose?

The instructor emphasizes that the "best" method depends entirely on the physical characteristics of the graph you are trying to represent.

  • Dense Graphs: Use Adjacency Matrix
    A graph is dense when almost every node connects to every other node (like a "Complete Graph"). In this case, use a Matrix. Using Linked Lists for a dense graph wastes memory because of the overhead required to store so many list pointers.
  • Sparse Graphs: Use Adjacency List
    A graph is sparse when there are very few connections compared to the number of vertices. If you use a Matrix here, the grid will be mostly filled with 0s. This is a massive waste of memory. An Adjacency List is dynamic and only allocates memory for the edges that actually exist.
