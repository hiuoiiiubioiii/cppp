Introduction to Stack

Data Structures & Algorithms Breakdown

🧠 1. What is a Stack?

Unlike Arrays (which allow random access) and Linked Lists (which allow sequential access), a Stack is a linear data structure with restricted access.

It is an ordered list, collection, or container of elements (of the same data type) that follows a strict rule for how data is inserted and deleted.

The Golden Rule: Insertion and Deletion can only happen from ONE end.

📦 2. Real-Life Examples & The Core Principle

To understand the rule, imagine a CD spindle or a stack of cafeteria plates. You can only place a new plate on the top. If you want a plate from the bottom, you must remove every plate above it first.

LIFO

Last In, First Out

(The last plate you put down is the first one you pick up)

FILO

First In, Last Out

(The first plate you put down is trapped until the end)

📐 3. Logical Representation

In Computer Science, we logically represent a stack as a vertical container with only one opening at the top. Data enters and exits through this exact same opening.

Insert (Push) ↓ ↑ Delete (Pop)
top
Data 3
Data 2
Data 1

Notice: The bottom is sealed. The pointer named top always tracks the highest piece of data in the container.

⚙️ 4. Fundamental Operations

We manipulate the stack using specific built-in functions. (Note: You can only insert elements of similar data types, e.g., all integers or all characters).

  • Push(x) : Inserts element 'x' onto the top of the stack.
  • Pop() : Removes the topmost element.
    Notice it takes no arguments (like 'x') because you don't get to choose what to remove; it's always the top item!
  • Peek() or top() : Returns the value of the top element without removing it from the stack.
  • isEmpty() : Returns True if the stack has zero elements, otherwise False.
  • isFull() : Returns True if the stack has reached its maximum capacity.

*Less common operations she mentioned: Search, Traverse, Find Min, Find Max.

💾 5. Memory Allocation

Before using a stack, you must allocate memory to it. There are two primary ways to build a stack in programming:

1. Static Implementation

Built using Arrays. The size is fixed and must be defined ahead of time (e.g., Size = 5).

2. Dynamic Implementation

Built using Linked Lists. The size is flexible and can grow or shrink as the program runs.

🔥 Key Exam Concepts:

  • Underflow: Attempting to Pop() when top == -1.
  • Overflow: Attempting to Push() when top == size - 1.
  • Garbage Data: When you Pop, you don't actually have to delete the data in memory. You just lower the top pointer. If you Push again, the new data simply overwrites the old garbage data.

How "Reversing a String" works:

If you have the string "abcd", you iterate through it and push each letter into the stack. 'a' goes to the bottom, and 'd' sits at the top. When you Pop() them out one by one, they naturally come out in the order: 'd', 'c', 'b', 'a'.
