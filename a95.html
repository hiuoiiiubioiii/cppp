Linear Search Algorithm

Also known as Sequential Search. A comprehensive, vertical study guide.

1. Introduction & Array Setup

Linear Search is a method for finding a target value within a list. It is called Sequential Search because it sequentially checks each element of the list until a match is found or the whole list has been searched.

2. The Working Principle (Tracing)

The algorithm starts at the 0th index and compares the target data (42) with each element, one by one. Watch the animation below to see how the computer executes this.

Initializing search...
Two Stopping Conditions: You stop searching the array under exactly two conditions:
  1. You find the data: You hit the target element and stop immediately.
  2. You reach the end: If the data isn't in the array (e.g., searching for 41), you check every single box. Once you reach the end, you stop.

3. The Code & The "break" Statement

We use an iterative approach (a for loop) to execute the sequential search. Let's look at the primary logic.

for (int i = 0; i < n; i++) 
{
    if (a[i] == data) 
    {
        printf("Element found at index: %d", i);
        break; // Crucial statement!
    }
}

Why is break; essential?

If we search for 42, we find it at index 5. If we don't include a break;, the loop will waste processing power by unnecessarily continuing to check index 6 and 7. The break; forces the program to instantly exit the loop.

4. Handling the "Not Found" Scenario

What if we search for a number that isn't there (like 41)? The loop runs from index 0 to 7 and terminates. The instructor provided two different methods to print "Element not found". (You only need to use one of these on an exam).

Method 1: Index Check (Checking the Loop Counter)

If the loop searches the whole array and never hits the break, the loop finishes naturally when i becomes equal to n (8). So, if i == n, the element was missing.

// Placed outside, below the for loop
if (i == n) 
{
    printf("Element not found");
}

Method 2: Using a Flag Variable

We create a variable called found and set it to 0. If we find the item, we change it to 1. After the loop, if it's still 0, the item doesn't exist.

int found = 0; // Initialize flag

for (int i = 0; i < n; i++) {
    if (a[i] == data) {
        printf("Element found at index: %d", i);
        found = 1; // Change flag
        break; 
    }
}

// Check flag after the loop
if (found == 0) {
    printf("Element not found");
}

5. Time Complexity Analysis

Time complexity is determined by the number of comparisons made.

1. Best Case: O(1)

The element you are looking for is at the very first index (e.g., searching for 15). It takes exactly 1 comparison. This is constant time: O(1).

2. Worst Case: O(n)

The element is at the very last index (e.g., 17), or it's not in the array at all. You must check every single element, resulting in n comparisons. This is linear time: O(n).

3. Average Case: O(n)

The element could be anywhere. To find the average, we add up the comparisons for all possible successful cases, and divide by the number of cases (n).

  • If found at index 0 = 1 comparison.
  • If found at index 1 = 2 comparisons... up to n.
Sum of all cases = 1 + 2 + 3 + ... + n

Using the mathematical formula for the sum of the first n natural numbers, the sum is:

Sum =
n(n + 1)2

Now, divide by the number of cases (n) to get the average:

Average =
n(n + 1) / 2n
=
n + 12

In Big-O notation, we drop constants (like the +1 and the /2). Therefore, the average time complexity is O(n).
