Types of Hashing & Collisions

Continuous Study Feed • Scroll to learn

1. Why Hashing?

Data processing relies heavily on storing and retrieving data efficiently. Traditional search methods depend on \(n\) (the number of elements):

  • O(n) Linear Search (Worst case)
  • O(log n) Binary Search

The Solution is Hashing

Hashing is a searching/storing technique designed to achieve a constant time complexity of O(1), meaning the search time does not depend on the number of elements.

2. The Mechanics & Collisions

To perform hashing, you need a Hash Table (an array of size \(m\)) and a Hash Function \(h(k)\). The most common method is the Division Method:

\[ \text{Location} = h(k) \pmod m \]

What is a Collision?

A collision occurs when the hash function generates the exact same index for two different keys.

// Example: Hash table size m = 10

Key = 6  → 6 % 10 = 6 (Store at index 6)

Key = 26 → 26 % 10 = 6 ← COLLISION! Index 6 is full.

3. Classification of Hashing

Open Hashing

Also called: Closed Addressing

  • Uses Chaining.
  • Elements can be stored outside the main array using Linked Lists.

Closed Hashing

Also called: Open Addressing

  • Elements must be stored strictly inside the hash table array.
  • 1. Linear Probing
  • 2. Quadratic Probing
  • 3. Double Hashing

7. Formal Mathematical Definition

While visually searching "the next empty spot" is easy, university exams require you to know the formal Linear Probing formula:

\[ (u + i) \pmod m \]
u = Original calculated index
i = Probe attempt (0, 1, 2...)
m = Hash table size

Formula Proof (Using Key 11 from earlier):

  • // u = 5, m = 10
  • i = 0 (5 + 0) % 10 = 5 -> Collision (Location full)
  • i = 1 (5 + 1) % 10 = 6 -> Empty! Successfully stored.
