Array Representation of Binary Trees

Sequential Representation • Scrollable Study Guide

🧠 1. Introduction & Prerequisites

Before diving into advanced sorting algorithms like Heap Sort, we must first understand how to store a binary tree inside a computer's memory.

What is a Binary Tree?
A tree data structure where each node has at most two children (either 0, 1, or 2 children).

The Core Problem: A tree is a 2D visual structure. A computer's memory (like an array) is a flat, 1D sequential structure. How do we compress the 2D tree into a 1D array?

The Solutions:

  • Arrays (Sequential Representation) — The focus of this lesson.
  • Linked Lists (Dynamic Node Representation) — Uses pointers.

🧮 3. Finding Relationships (Formulas)

If you look at the flat array above, you can no longer visually "see" who is the parent or the child. We use mathematical formulas based on the array's index (i) to find these relationships.

Case 1: 0-Indexed Array (Array starts at 0)

If a node is located at index i:

Left Child = (2 * i) + 1
Right Child = (2 * i) + 2
Parent = ⌊ (i - 1) / 2 ⌋ // Floor function (round down)

Example test: Find the parent of node E.
In our array, E is at index i = 4.
Parent = (4 - 1) / 2 = 3 / 2 = 1.5. Round down = 1.
Index 1 is B, which is correct!

Case 2: 1-Indexed Array (Array starts at 1)

If a node is located at index i:

Left Child = 2 * i
Right Child = (2 * i) + 1
Parent = ⌊ i / 2 ⌋

⚠️ 4. The "Complete Binary Tree" Trap

This is where many students fail their exams. What if the tree is not perfectly filled out?

If you just read the letters level-by-level left-to-right, the formulas break. To fix this, you must treat the tree as a Complete Binary Tree by inserting "blank" or "dummy" nodes to fill in the missing gaps.

Scenario: Node D is missing children. H and I belong to E.

Correct Array Representation:

C2
D3
E4

G6
_7
_8

I10
By adding the two blank nodes, H and I shift to indices 9 and 10. If you check E's left child (index 4): (2*4)+1 = 9. The math works perfectly again!

📉 5. Drawbacks of Array Representation

Arrays are incredibly memory-efficient for complete or nearly complete trees (which is why they are perfect for Heaps). However, if a tree is heavily skewed, arrays waste a massive amount of memory.
