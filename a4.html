Arrays & Pointers in C

Masterclass Breakdown: Memory tapes, Pointer Arithmetic, and the Golden Rules

1. How Arrays are Stored in Memory

When you declare an array, the compiler allocates contiguous (side-by-side) blocks of memory.

int a[5] = {6, 2, 1, 5, 3};

The Audio Context: If you initialize the array immediately like above, writing the size [5] is actually optional. The compiler figures it out automatically!

Visualizing the "Tape of Bytes"

Assuming a typical compiler where an int takes 4 bytes, and assuming the computer starts storing our array at memory address 100, here is what memory looks like:

a[0] 6 100
a[1] 2 104
a[2] 1 108
a[3] 5 112
a[4] 3 116
💡 Notice: The addresses jump by 4 (100 → 104 → 108). This is because every integer requires exactly 4 bytes of space.

2. Pointer Basics Refresher

Before linking arrays to pointers, we must remember how standard pointers work.

int *p;    // Pointer to an integer
float *q;  // Pointer to a float
char *c;   // Pointer to a character
⚠️ Crucial Audio Concept: The data type of a pointer itself is not an integer or a float. A pointer always stores a memory address (a long hexadecimal number). The int simply tells the compiler the data type of the variable that lives at that address.

Valid vs. Invalid Operations

Assume we have int b = 10; stored at address 200, and a pointer int *p;.

✅ p = &b;
Valid: This reads as "p equals the address of b". p now correctly holds the address `200`.
❌ p = b;
Invalid: A pointer cannot hold a standard integer value like 10. It strictly expects memory addresses.

How to Print Them

  • printf("%d", b); → Prints 10
  • printf("%p", &b); → Prints 200 (Address of b)
  • printf("%p", p); → Prints 200 (Because p holds the address of b)
  • printf("%d", *p); → Prints 10 (* is the Dereferencing Operator. "Give me the value at the address p is holding")

3. The Golden Rule: Arrays ARE Pointers

Here is the most important concept in the lecture: The name of an array acts as a pointer to its base address.

If you write printf("%p", a); it will output 100 (the exact same thing as writing &a[0]).

Assigning Arrays to Pointer Variables

Because a is essentially an address, you can assign it directly to a pointer without needing the & symbol.

int *q;
q = a;        // ✅ VALID. q now holds 100.
q = &a[2];    // ✅ VALID. q now holds 108.
q = a[2];     // ❌ INVALID. You can't put the value '1' into a pointer.

4. Pointer Arithmetic

You can add or subtract integers from pointers, but the math is different from standard algebra. It is based on the size of the data type.

The Math behind q++

If q currently holds 100 (pointing to a[0]), and you write q++ (which means q = q + 1):

  • Normal Math: 100 + 1 = 101
  • Pointer Math: 100 + (1 * size of data type).
  • Because integers are 4 bytes: 100 + (1 * 4) = 104.

q now points to a[1]. (Note: If it were a character pointer, the size is 1 byte, so 100 + 1 would equal 101).

The Fatal Error: a++

❌ a++; is strictly prohibited!

Why? q is a pointer variable; you can change what it points to. But a (the array name) is a constant pointer. Once the array is created in memory, you cannot physically move its base address.

5. The Syntax "Cheat Sheet"

The instructor writes out different ways to access the exact same array elements. Assuming int *q = a;, memorize these equivalents:

1. Getting Addresses (e.g., pointing to the 3rd element: 108)

  • Standard Array Format: &a[2]
  • Pointer Arithmetic on Array Name: (a + 2)
  • Using Pointer Variable: (q + 2)

2. Getting Actual Values (e.g., getting the value: 1)

  • Standard Array Format: a[2]
  • Dereferencing Array Name: *(a + 2)
  • Dereferencing Pointer Variable: *(q + 2)
  • Index notation on a Pointer: q[2]

3. The Weird "C Language" Quirk

Because addition is commutative (a + i is the exact same as i + a), the C compiler allows a very strange syntax:

a[i] == *(a + i) == *(i + a) == i[a]

Yes, writing 2[a] is perfectly valid C code and will successfully print the value 1.

6. Practical Code Example

How does this look in a real program? Here is how to use pointers inside for loops to read user input and print it back to the screen.

int a[5];       // Declare array
int *q = a;     // Create pointer q, pointing to base address of a (100)
int i;

// 1. TAKING INPUT FROM USER
for(i = 0; i < 5; i++) {
    // Normal way: scanf("%d", &a[i]);
    
    // Pointer way using array name:
    scanf("%d", (a + i)); 
    
    // OR you could write: scanf("%d", (q + i));
}

// 2. PRINTING VALUES TO SCREEN
for(i = 0; i < 5; i++) {
    // Normal way: printf("%d", a[i]);
    
    // Pointer way using pointer variable:
    printf("%d ", *(q + i)); 
    
    // OR you could write: printf("%d ", *(a + i));
}
🎓 Summary: Always remember that values require asterisks * or brackets []. Addresses require ampersands & or bare pointers/array names.
